# 微信小程序网络请求配置

## 问题现象
- 浏览器访问 `http://localhost:8080/api/checkin/statistics/user` 返回 401 ✅
- 小程序访问同样的接口返回 404 ❌

## 原因分析
微信小程序对网络请求有严格的限制，localhost请求可能被拦截。

## 解决方案

### 方案1：开启"不校验合法域名"（推荐用于开发）

在微信开发者工具中：

1. 点击右上角"详情"按钮
2. 找到"本地设置"标签页
3. 勾选以下选项：
   - ✅ **不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书**
   - ✅ **不校验安全域名、TLS 版本以及 HTTPS 证书**
4. 重新编译项目

### 方案2：使用真实IP地址

如果方案1不生效，修改前端的BASE_URL：

**文件**：`couple-fitness-weapp/utils/request.js`

```javascript
// 修改前
const BASE_URL = 'http://localhost:8080';

// 修改后（使用本机IP）
const BASE_URL = 'http://192.168.x.x:8080';  // 替换为你的本机IP
```

**获取本机IP**：
- Windows: 打开cmd，输入 `ipconfig`，查看IPv4地址
- Mac/Linux: 打开终端，输入 `ifconfig` 或 `ip addr`

### 方案3：配置后端允许跨域

确保后端已配置CORS，允许小程序访问。

**检查文件**：`RuoYi-Vue/ruoyi-framework/src/main/java/com/ruoyi/framework/config/CorsConfig.java`

应该包含类似配置：
```java
@Configuration
public class CorsConfig {
    @Bean
    public CorsFilter corsFilter() {
        CorsConfiguration config = new CorsConfiguration();
        config.setAllowCredentials(true);
        config.addAllowedOriginPattern("*");
        config.addAllowedHeader("*");
        config.addAllowedMethod("*");
        // ...
    }
}
```

## 验证步骤

### 1. 检查开发者工具设置
确认"不校验合法域名"已勾选

### 2. 清除缓存
- 点击"清缓存" → "清除全部缓存"
- 重新编译

### 3. 查看网络请求
- 打开"调试器" → "Network"标签
- 重新登录
- 查看请求的URL和响应

### 4. 查看控制台日志
查看是否还有404错误，应该看到：
```
【网络请求】发起请求: http://localhost:8080/api/checkin/statistics/user
【网络请求】请求成功: { statusCode: 200, ... }
```

## 常见问题

### Q1: 勾选了"不校验合法域名"仍然404
**A**: 
1. 确保后端服务正在运行
2. 尝试使用IP地址替代localhost
3. 检查防火墙是否阻止了请求

### Q2: 使用IP地址后仍然失败
**A**: 
1. 确保手机/模拟器和电脑在同一网络
2. 检查IP地址是否正确
3. 尝试关闭防火墙测试

### Q3: 真机调试时无法访问
**A**: 
1. 真机必须使用HTTPS或配置合法域名
2. 开发阶段建议使用模拟器
3. 或者部署后端到有HTTPS的服务器
