# 隐私设置打卡记录查看权限修复

## 问题描述
用户在隐私设置中关闭"伴侣可查看打卡记录"开关后,伴侣仍然可以看到打卡记录。

## 问题原因
后端 `CheckInController.listCheckInByUserId()` 方法没有检查隐私设置,直接返回了打卡记录列表。

## 修复内容

### 1. 后端修复 (CheckInController.java)

**添加依赖注入:**
```java
@Autowired
private com.ruoyi.web.service.IPrivacySettingService privacySettingService;

@Autowired
private com.ruoyi.web.mapper.PartnershipMapper partnershipMapper;
```

**修改 `listCheckInByUserId()` 方法:**
- 验证是否是伴侣关系
- 检查隐私设置 `allowPartnerViewCheckins`
- 如果隐私设置关闭,返回空列表

**修改 `getPartnerStatistics()` 方法:**
- 检查隐私设置 `allowPartnerViewStats`
- 如果隐私设置关闭,返回错误提示

### 2. 前端优化 (partner-checkins/index.js)

在 `loadCheckInList()` 方法中添加提示:
- 当返回空数据时,显示"对方已关闭打卡记录查看权限"

## 测试步骤

1. 用户A关闭"伴侣可查看打卡记录"
2. 用户B访问用户A的打卡记录页面
3. 应该看到空列表和提示信息
4. 用户A重新开启权限
5. 用户B刷新页面,应该能看到打卡记录
