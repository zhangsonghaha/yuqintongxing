# 测试后端接口

## 使用浏览器测试

### 1. 测试服务器是否运行
打开浏览器，访问：
```
http://localhost:8080
```

**预期结果**：
- 如果显示页面或返回数据：✅ 服务器正在运行
- 如果无法访问：❌ 服务器未启动

### 2. 测试API接口（无需登录）
访问以下URL（这些接口应该返回401，说明接口存在但需要登录）：

```
http://localhost:8080/api/checkin/statistics/user
http://localhost:8080/api/checkin/statistics/partner
http://localhost:8080/api/checkin/recent?limit=3
```

**预期结果**：
- 返回 `{"code":401,"msg":"未授权"}` 或类似信息：✅ 接口存在
- 返回 404：❌ 接口不存在或路径错误
- 无法连接：❌ 服务器未启动

```

```

```
Authorization: Bearer YOUR_TOKEN_HERE
```

## 常见问题排查

### 问题1：404 Not Found
**可能原因**：
- Controller类没有被Spring扫描到
- @RequestMapping路径配置错误
- 方法上缺少@GetMapping/@PostMapping注解

**解决方法**：
检查 `CheckInController.java`：
```java
@RestController
@RequestMapping("/api/checkin")  // 确保这个路径正确
public class CheckInController extends BaseController {
    
    @GetMapping("/statistics/user")  // 完整路径：/api/checkin/statistics/user
    public AjaxResult getUserStatistics() {
        // ...
    }
}
```

### 问题2：401 Unauthorized
**说明**：接口存在，但需要登录认证（这是正常的）

**解决方法**：
- 先调用登录接口获取token
- 在后续请求中携带token

### 问题3：500 Internal Server Error
**可能原因**：
- 数据库连接失败
- 业务逻辑错误
- 缺少必要的数据

**解决方法**：
- 查看后端控制台的错误堆栈
- 检查数据库连接
- 检查数据库中是否有必要的数据

## 快速验证脚本

创建一个 `test-backend.sh` 文件：
```bash
#!/bin/bash

echo "测试后端服务..."

# 测试服务器
echo "1. 测试服务器是否运行..."
curl -s http://localhost:8080 > /dev/null
if [ $? -eq 0 ]; then
    echo "✅ 服务器正在运行"
else
    echo "❌ 服务器未启动"
    exit 1
fi

# 测试API接口
echo "2. 测试API接口..."
response=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/api/checkin/statistics/user)
if [ "$response" = "401" ] || [ "$response" = "200" ]; then
    echo "✅ API接口存在 (HTTP $response)"
else
    echo "❌ API接口不存在 (HTTP $response)"
fi

echo "测试完成！"
```

运行：
```bash
chmod +x test-backend.sh
./test-backend.sh
```
