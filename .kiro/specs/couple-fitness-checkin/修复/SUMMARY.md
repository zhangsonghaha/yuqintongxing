## 🏗️ 技术架构

### 前端
- **框架**: WeChat Mini Program (原生)
- **语言**: JavaScript ES6+
- **UI 库**: Vant Weapp v1.10.4+
- **状态管理**: getApp() + wx.setStorageSync()
- **网络**: wx.request() + 自定义包装器

### 后端
- **框架**: Spring Boot 2.5.15
- **语言**: Java 8+
- **ORM**: MyBatis + PageHelper
- **认证**: Spring Security 5.7.14 + JWT
- **数据库**: MySQL 8.0+
- **连接池**: Druid 1.2.27

### 部署
- **前端**: 微信小程序平台
- **后端**: Docker + Kubernetes (可选)
- **数据库**: MySQL 8.0+ (主从复制)
- **缓存**: Redis (可选)
- **CDN**: 静态资源加速

- 基于角色的访问控制 (RBAC)

- HTTPS 传输加密
- 敏感数据存储加密
- 敏感词过滤

### 隐私保护
- 用户只能访问自己的数据
- 用户只能访问配对伴侣的数据
- 支持隐私设置
- 账号注销时完全删除数据

## 🎓 关键决策说明

### 为什么选择原生微信小程序？
- 简化开发流程，减少依赖
- 更好的性能和用户体验
- 直接使用微信 API，无需转译

### 为什么不包含虚拟宠物？
- MVP 阶段优先核心功能
- 虚拟宠物复杂度高，可在 V1.0 实现
- 先验证用户需求，再投入资源

### 为什么使用 RuoYi-Vue 框架？
- 快速开发，内置权限管理
- 包含代码生成工具，提高效率
- 成熟稳定，社区支持好

### 为什么选择 MySQL？
- 成熟稳定，性能优异
- 支持 JSON 类型，灵活存储
- 支持主从复制，易于扩展
