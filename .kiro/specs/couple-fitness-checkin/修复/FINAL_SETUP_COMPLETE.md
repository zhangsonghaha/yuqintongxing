## 💡 提示

1. **数据隔离**: 手机号登录和微信登录是两个独立的账号系统
   - 手机号 `13800138000` 的用户ID: `phone_13800138000`
   - 微信用户的用户ID: `test_user_` + 微信昵称

2. **测试多用户**: 
   - 使用不同的手机号可以创建不同的测试账号
   - 使用不同的微信账号也会创建不同的用户

3. **数据刷新**: 
   - 主页的 `onShow()` 方法会在每次显示时刷新数据
   - 打卡后返回主页应该会自动更新统计数据

4. **清除数据**: 
   - 如果需要清除测试数据，可以直接删除数据库中的记录
   - 或者使用不同的手机号/微信账号测试

## 🐛 如果遇到问题

### 问题1: 手机号登录失败
- 检查后端是否已重启
- 查看后端控制台是否有错误日志
- 确认验证码输入的是 `123456`

### 问题2: 看不到打卡记录
- 检查控制台日志，查看 API 返回的数据
- 确认使用的是相同的登录方式和账号
- 查看数据库中是否有对应的记录

### 问题3: 主页数据不更新
- 检查 `onShow()` 方法是否被调用
- 查看控制台日志中的"加载数据"相关信息
- 尝试手动刷新（下拉刷新）

### 问题4: 编译错误
- 确认 app.json 格式正确
- 确认所有页面文件都已创建
- 尝试清除缓存后重新编译
