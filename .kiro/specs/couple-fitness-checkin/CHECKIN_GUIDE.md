# 打卡流程指南

## 📱 打卡步骤

### 第一步: 进入首页
1. 打开小程序
2. 登录成功后进入首页
3. 看到"打卡(3秒完成)"按钮

### 第二步: 点击打卡按钮
1. 点击首页中间的粉红色"打卡(3秒完成)"按钮
2. 系统会检查今天是否已打卡
3. 如果未打卡,跳转到打卡页面

### 第三步: 填写打卡信息

#### 选择运动类型
- 点击"运动类型"下拉框
- 从列表中选择运动类型:
  - 跑步
  - 游泳
  - 骑行
  - 瑜伽
  - 健身
  - 篮球
  - 足球
  - 羽毛球
  - 乒乓球
  - 散步

#### 输入运动时长
- 在"运动时长"输入框中输入分钟数
- 例如: 30 (表示 30 分钟)
- 系统会自动计算卡路里

#### 上传打卡照片(可选)
- 点击"拍照"按钮: 使用相机拍照
- 或点击"相册"按钮: 从相册选择照片
- 系统会自动压缩图片
- 上传成功后显示预览

#### 添加备注(可选)
- 在"备注"框中输入运动感受
- 例如: "今天天气很好,跑步很舒服"

### 第四步: 提交打卡
1. 检查所有必填项是否已填写:
   - ✅ 运动类型 (必填)
   - ✅ 运动时长 (必填)
   - ⭕ 打卡照片 (可选)
   - ⭕ 备注 (可选)

2. 点击"确认打卡"按钮
3. 系统验证数据
4. 显示"打卡成功"提示
5. 自动返回首页

## ⚠️ 常见问题

### Q1: 点击打卡按钮没反应

**可能原因:**
- 页面还在加载中
- 网络连接不稳定
- 打卡页面路径配置错误

**解决方案:**
1. 检查网络连接
2. 等待页面完全加载
3. 重新打开小程序
4. 检查 app.json 中是否注册了打卡页面

### Q2: 打卡页面加载失败

**可能原因:**
- 打卡页面文件缺失
- 页面路径配置错误
- 小程序编译错误

**解决方案:**
1. 确保 `pages/checkin/` 目录存在
2. 确保 `app.json` 中注册了打卡页面:
   ```json
   "pages": [
     "pages/checkin/index"
   ]
   ```
3. 重新编译小程序

### Q3: 提交打卡失败

**可能原因:**
- 后端服务未启动
- API 地址配置错误
- 网络连接问题
- 用户未登录

**解决方案:**
1. 确保后端服务正在运行
2. 检查 `app.js` 中的 `apiBaseUrl` 配置
3. 检查网络连接
4. 确保已登录

### Q4: 卡路里计算不准确

**说明:**
- 卡路里是基于平均值的估算
- 实际消耗取决于个人体质、速度等因素
- 这是简单的参考值

**改进方案:**
- 后续版本可以添加用户体质信息
- 根据用户身高、体重、年龄等计算更精确的值

### Q5: 图片上传失败

**可能原因:**
- 文件大小超过 5MB
- 文件格式不支持
- 网络连接不稳定

**解决方案:**
1. 确保图片大小 < 5MB
2. 支持的格式: jpg, jpeg, png, gif
3. 检查网络连接
4. 重新选择图片
