# 配对页面入口说明

## 页面路径
配对页面位于: `couple-fitness-weapp/pages/partnership/index`

## 进入方式

### 方式 1: 从首页进入 (推荐)
1. 打开小程序,进入首页
2. 如果用户还没有配对伴侣,在"对方的状态"区域会显示:
   - 提示文字: "还没有配对伴侣"
   - 按钮: "去配对"
3. 点击"去配对"按钮,即可跳转到配对页面

### 方式 2: 从个人中心进入
1. 打开小程序,点击底部导航栏的"我的"标签
2. 在个人中心页面,找到"情侣功能"区域
3. 点击"情侣配对"菜单项
4. 即可跳转到配对页面

### 方式 3: 直接导航 (开发调试)
在微信开发者工具中,可以直接输入路径:
```
/pages/partnership/index
```

## 配对页面功能

### 三个标签页

#### 1. 生成邀请码
- 点击"生成邀请码"按钮
- 系统生成6位随机邀请码
- 显示邀请码和二维码
- 可以复制邀请码或保存二维码
- 分享给伴侣使用

#### 2. 输入邀请码
- 输入伴侣分享的6位邀请码
- 点击"发送配对请求"
- 等待伴侣接受

#### 3. 待接受请求
- 查看收到的配对请求列表
- 显示请求者的头像和昵称
- 可以选择"接受"或"拒绝"
- 接受后即完成配对

## 已配对状态
如果用户已经配对:
- 页面会显示当前伴侣信息
- 包括伴侣头像、昵称、配对时间
- 提供"解除配对"按钮

## 技术实现

### 前端文件
- `couple-fitness-weapp/pages/partnership/index.wxml` - 页面结构
- `couple-fitness-weapp/pages/partnership/index.wxss` - 页面样式
- `couple-fitness-weapp/pages/partnership/index.js` - 页面逻辑
- `couple-fitness-weapp/pages/partnership/index.json` - 页面配置

### 后端 API
- `POST /api/partnership/generate-invite` - 生成邀请码
- `GET /api/partnership/invite-info` - 获取邀请码信息
- `POST /api/partnership/request` - 发送配对请求
- `POST /api/partnership/accept/{id}` - 接受配对请求
- `POST /api/partnership/reject/{id}` - 拒绝配对请求
- `GET /api/partnership/pending` - 获取待接受请求列表
- `GET /api/partnership/partner` - 获取当前伴侣信息
- `DELETE /api/partnership/dissolve` - 解除配对关系

## 注意事项

1. **登录状态**: 用户必须先登录才能访问配对页面
2. **邀请码有效期**: 邀请码有效期为24小时
3. **唯一配对**: 一个用户同时只能与一个伴侣配对
4. **解除配对**: 解除配对后,双方都需要重新配对

## 测试流程

### 测试配对功能
1. 准备两个测试账号 (用户A 和 用户B)
2. 用户A 登录,进入配对页面,生成邀请码
3. 用户B 登录,进入配对页面,输入用户A的邀请码
4. 用户A 在"待接受请求"标签页查看请求
5. 用户A 点击"接受",完成配对
6. 双方返回首页,可以看到对方的打卡状态

### 测试解除配对
1. 已配对的用户进入配对页面
2. 点击"解除配对"按钮
3. 确认解除操作
4. 配对关系解除,双方恢复未配对状态
