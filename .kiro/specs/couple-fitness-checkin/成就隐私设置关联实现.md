# 成就隐私设置关联实现

## 需求说明

用户反馈：
1. 伴侣主页的成就没有显示
2. 成就显示应该与"隐私与安全"页面的设置关联，根据用户的隐私设置决定是否可以查看对方的成就

## 现有功能

小程序已经有"隐私与安全"页面（`pages/privacy-setting/`），包含以下隐私开关：
- `allowPartnerViewCheckins` - 伴侣可查看打卡记录
- `allowPartnerViewStats` - 伴侣可查看统计数据
- `allowPartnerViewAchievements` - 伴侣可查看成就

## 实现方案

### 1. 后端修改

#### 成就控制器（AchievementController.java）

在 `getUserAchievements` 方法中添加隐私检查逻辑：

```java
@GetMapping("/user/{userId}")
public AjaxResult getUserAchievements(@PathVariable Long userId) {
    Long currentUserId = SecurityUtils.getUserId();
    
    // 检查隐私设置：目标用户是否允许查看成就
    // 注意：查询目标用户（userId）的隐私设置，而不是当前用户的
    try {
        // 调用隐私设置服务检查权限
        boolean allowView = privacySettingsService.checkAchievementViewPermission(userId, currentUserId);
        if (!allowView) {
            return AjaxResult.error("对方已设置隐私，无法查看成就");
        }
    } catch (Exception e) {
        log.warn("检查隐私设置失败，默认允许查看: {}", e.getMessage());
    }
    
    List<Achievement> achievements = achievementService.getUserAchievements(userId);
    return AjaxResult.success(achievements);
}
```

**注意**：
- 需要创建 `PrivacySettingsService` 服务来处理隐私设置的查询和权限检查
- 需要创建对应的 Mapper 和 XML 映射文件
- 隐私设置数据存储在 `privacy_settings` 表中（已创建 SQL 文件）

### 2. 前端修改

#### 伴侣主页 JS（partner-profile/index.js）

**添加数据字段**：
```javascript
data: {
  achievementsHidden: false, // 成就是否因隐私设置被隐藏
  // ... 其他字段
}
```

**修改成就数据获取逻辑**：
```javascript
let achievementsHidden = false;

try {
  const achievementsResponse = await achievementAPI.getAchievementsByUserId(this.data.partnerId);
  
  // 检查是否因为隐私设置被拒绝
  if (achievementsResponse && achievementsResponse.code !== 200) {
    if (achievementsResponse.msg && achievementsResponse.msg.includes('隐私')) {
      achievementsHidden = true;
    }
  } else if (achievementsResponse && achievementsResponse.code === 200) {
    achievements = achievementsResponse.data || [];
  }
} catch (error) {
  // 检查错误信息是否包含隐私相关
  if (error.message && error.message.includes('隐私')) {
    achievementsHidden = true;
  }
}

this.setData({
  achievements,
  achievementsHidden
});
```

## 测试场景

### 场景1：成就可见（默认）
1. 用户A和用户B配对
2. 用户B的隐私设置中"伴侣可查看成就"开关为开启状态
3. 用户A访问用户B的主页
4. **预期结果**：可以看到用户B的成就列表

### 场景2：成就隐藏
1. 用户A和用户B配对
2. 用户B在"隐私与安全"页面关闭"伴侣可查看成就"开关
3. 用户A访问用户B的主页
4. **预期结果**：
   - 成就标签页显示隐私提示
   - 提示文字："对方已设置隐私，无法查看成就"
   - 提示说明："可以提醒TA在'隐私与安全'中开启"

### 场景3：无成就数据
1. 用户A和用户B配对
2. 用户B的隐私设置允许查看成就
3. 用户B确实没有解锁任何成就
4. 用户A访问用户B的主页
5. **预期结果**：显示"暂无成就数据"

## 数据库表结构

```sql
CREATE TABLE IF NOT EXISTS `privacy_settings` (
  `setting_id` BIGINT(20) NOT NULL AUTO_INCREMENT COMMENT '设置ID',
  `user_id` BIGINT(20) NOT NULL COMMENT '用户ID',
  `show_achievements` TINYINT(1) DEFAULT 1 COMMENT '是否显示成就（1=显示，0=隐藏）',
  `show_statistics` TINYINT(1) DEFAULT 1 COMMENT '是否显示统计数据（1=显示，0=隐藏）',
  `show_checkin_records` TINYINT(1) DEFAULT 1 COMMENT '是否显示打卡记录（1=显示，0=隐藏）',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`setting_id`),
  UNIQUE KEY `uk_user_id` (`user_id`),
  KEY `idx_user_id` (`user_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户隐私设置表';
```

## API 接口设计

### 获取隐私设置
```
GET /api/privacy
Authorization: Bearer {token}

Response:
{
  "code": 200,
  "msg": "操作成功",
  "data": {
    "allowPartnerViewCheckins": true,
    "allowPartnerViewStats": true,
    "allowPartnerViewAchievements": true,
    "dataShareScope": 1
  }
}
```

### 更新隐私设置
```
PUT /api/privacy
Authorization: Bearer {token}
Content-Type: application/json

{
  "allowPartnerViewCheckins": true,
  "allowPartnerViewStats": true,
  "allowPartnerViewAchievements": false,
  "dataShareScope": 1
}

Response:
{
  "code": 200,
  "msg": "保存成功"
}
```

### 获取指定用户成就（带隐私检查）
```
GET /api/achievement/user/{userId}
Authorization: Bearer {token}

Response (允许查看):
{
  "code": 200,
  "msg": "操作成功",
  "data": [
    {
      "achievementId": 1,
      "badgeName": "首次打卡",
      "unlocked": true,
      ...
    }
  ]
}

Response (隐私限制):
{
  "code": 403,
  "msg": "对方已设置隐私，无法查看成就",
  "data": null
}
```

## 注意事项

1. **隐私检查逻辑**：
   - 检查的是目标用户（被查看者）的隐私设置
   - 不是当前用户（查看者）的隐私设置

2. **默认行为**：
   - 如果隐私设置查询失败，默认允许查看（向后兼容）
   - 新用户注册时自动创建默认隐私设置（所有选项开启）

3. **错误处理**：
   - 前端需要区分"无数据"和"隐私限制"两种情况
   - 后端返回明确的错误码和提示信息

4. **用户体验**：
   - 隐私提示要友好，提供解决方案
   - 不要让用户感到困惑或受挫
