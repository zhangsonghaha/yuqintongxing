# 伴侣主页数据显示修复总结

## 问题描述

用户反馈伴侣主页存在以下问题：
1. 运动数据全部显示为 0（总卡路里、总时长、最长连续）
2. 成就列表为空，只显示一个灰色徽章图标

## 问题分析

### 1. 运动数据显示为 0 的原因

后端 `CheckInStatistics` 类缺少以下字段：
- `totalCalories`（总消耗卡路里）
- `totalDuration`（总运动时长）
- `maxConsecutiveDays`（最长连续打卡天数的别名）

前端伴侣主页需要显示这些数据，但后端统计 API 没有返回。

### 2. 成就数据为空的原因

前端调用了 `achievementAPI.getAchievementsByUserId(partnerId)` 获取伴侣成就，但可能：
- 后端接口返回数据格式不正确
- 伴侣确实没有解锁任何成就
- 成就系统尚未完全实现

## 修复方案

### 后端修复

#### 1. 更新 CheckInStatistics 类

**文件**: `RuoYi-Vue/ruoyi-admin/src/main/java/com/ruoyi/web/domain/CheckInStatistics.java`

添加字段：
```java
private BigDecimal totalCalories;       // 总消耗卡路里
private Integer totalDuration;          // 总运动时长（分钟）
private Integer maxConsecutiveDays;     // 最长连续打卡天数（别名）
private java.util.Map<String, Integer> exerciseTypeDistribution; // 运动类型分布
```

添加 getter/setter 方法，其中 `maxConsecutiveDays` 是 `longestConsecutiveDays` 的别名。

#### 2. 更新 CheckInMapper 接口

**文件**: `RuoYi-Vue/ruoyi-admin/src/main/java/com/ruoyi/web/mapper/CheckInMapper.java`

添加方法：
```java
BigDecimal selectTotalCalories(@Param("userId") Long userId);
Integer selectTotalDuration(@Param("userId") Long userId);
```

#### 3. 更新 MyBatis XML 映射

**文件**: `RuoYi-Vue/ruoyi-admin/src/main/resources/mapper/web/CheckInMapper.xml`

添加 SQL 查询：
```xml
<!-- 获取用户总消耗卡路里 -->
<select id="selectTotalCalories" resultType="java.math.BigDecimal">
    SELECT COALESCE(SUM(calories), 0) FROM check_in_record 
    WHERE user_id = #{userId}
    AND deleted_at IS NULL
</select>

<!-- 获取用户总运动时长 -->
<select id="selectTotalDuration" resultType="java.lang.Integer">
    SELECT COALESCE(SUM(duration), 0) FROM check_in_record 
    WHERE user_id = #{userId}
    AND deleted_at IS NULL
</select>
```

#### 4. 更新 CheckInServiceImpl

**文件**: `RuoYi-Vue/ruoyi-admin/src/main/java/com/ruoyi/web/service/impl/CheckInServiceImpl.java`

在 `getUserStatistics` 方法中添加：
```java
// 获取总卡路里和总时长
BigDecimal totalCalories = checkInMapper.selectTotalCalories(userId);
statistics.setTotalCalories(totalCalories);

Integer totalDuration = checkInMapper.selectTotalDuration(userId);
statistics.setTotalDuration(totalDuration);

// 获取运动类型分布
List<java.util.Map<String, Object>> distributionList = checkInMapper.selectExerciseTypeDistribution(userId);
java.util.Map<String, Integer> distributionMap = new java.util.HashMap<>();
for (java.util.Map<String, Object> item : distributionList) {
    String exerciseType = (String) item.get("exerciseType");
    Object countObj = item.get("count");
    Integer count = 0;
    if (countObj instanceof Long) {
        count = ((Long) countObj).intValue();
    } else if (countObj instanceof Integer) {
        count = (Integer) countObj;
    }
    distributionMap.put(exerciseType, count);
}
statistics.setExerciseTypeDistribution(distributionMap);
```

### 前端修复

#### 1. 修复 WXML 模板错误

**文件**: `couple-fitness-weapp/pages/partner-profile/index.wxml`

问题：
- 第 13 行使用了 `.substring()` 方法（WXML 不支持）
- 第 71 行使用了 `.toFixed()` 方法（WXML 不支持）

修复：
```xml
<!-- 修复前 -->
<text>{{(partnerInfo.partnerNickname || partnerInfo.nickname || '对方').substring(0, 1)}}</text>
<view class="distribution-fill" style="width: {{(item.count / stats.totalCheckIns * 100).toFixed(0)}}%"></view>

<!-- 修复后 -->
<text>{{partnerInfo.avatarText}}</text>
<view class="distribution-fill" style="width: {{item.percentage}}%"></view>
```

#### 2. 更新 JS 数据处理

**文件**: `couple-fitness-weapp/pages/partner-profile/index.js`

在 `loadData` 方法中：
```javascript
// 计算头像文字
partnerInfo.avatarText = partnerName.substring(0, 1);

// 处理运动类型分布
let exerciseDistribution = [];
if (stats.exerciseTypeDistribution && typeof stats.exerciseTypeDistribution === 'object') {
  exerciseDistribution = Object.entries(stats.exerciseTypeDistribution).map(([type, count]) => ({
    type: this.getExerciseTypeName(type),
    count: count,
    percentage: stats.totalCheckIns > 0 ? Math.round((count / stats.totalCheckIns) * 100) : 0
  }));
  exerciseDistribution.sort((a, b) => b.count - a.count);
}

// 设置数据
this.setData({
  stats: {
    totalCheckIns: stats.totalCheckIns || 0,
    consecutiveDays: stats.consecutiveDays || 0,
    maxConsecutiveDays: stats.maxConsecutiveDays || stats.longestConsecutiveDays || 0,
    totalDuration: stats.totalDuration || 0,
    totalCalories: Math.round(stats.totalCalories || 0)
  },
  exerciseDistribution
});
```

#### 3. 修复首页事件处理

**文件**: `couple-fitness-weapp/pages/index/index.js`

移除了不必要的 `e.stopPropagation()` 调用，因为在微信小程序中事件对象可能为 `undefined`。

**文件**: `couple-fitness-weapp/pages/index/index.wxml`

移除了重复的 `catchtap` 绑定，只保留 `bindtap`。

## 已知问题

### 1. 成就系统未完全实现

如果伴侣主页成就列表仍然为空，可能是因为：
- 成就系统尚未完全实现
- 用户确实没有解锁任何成就
- 后端 `GET /api/achievement/user/{userId}` 接口返回数据格式不正确

建议：
- 检查后端成就接口是否正常工作
- 确认数据库中是否有成就数据
- 测试成就解锁逻辑是否正常触发

### 2. 运动类型分布可能为空

如果用户只打卡过一次或没有打卡，运动类型分布会为空，这是正常的。
