# 互动通知功能实现方案

## 需求描述

当用户的打卡记录被伴侣点赞或评论时，应该在界面上收到提示通知。

## 功能场景

### 场景1：收到点赞通知
- 用户 A 点赞了用户 B 的打卡记录
- 用户 B 在首页或其他页面看到通知提示
- 点击通知可以查看被点赞的打卡记录

### 场景2：收到评论通知
- 用户 A 评论了用户 B 的打卡记录
- 用户 B 收到通知，显示评论内容预览
- 点击通知可以查看完整评论

### 场景3：通知列表
- 用户可以查看所有历史通知
- 区分已读和未读通知
- 显示通知时间和内容

## 技术方案

### 方案1：数据库轮询（推荐，简单实用）

**优点**：
- 实现简单
- 不需要额外的服务
- 适合小规模应用

**缺点**：
- 不是实时的（有延迟）
- 增加服务器负担

**实现步骤**：

#### 1. 数据库设计

创建通知表 `notification`：

```sql
CREATE TABLE `notification` (
  `notification_id` BIGINT(20) NOT NULL AUTO_INCREMENT COMMENT '通知ID',
  `user_id` BIGINT(20) NOT NULL COMMENT '接收通知的用户ID',
  `from_user_id` BIGINT(20) NOT NULL COMMENT '触发通知的用户ID',
  `type` VARCHAR(20) NOT NULL COMMENT '通知类型：like-点赞，comment-评论',
  `record_id` BIGINT(20) NOT NULL COMMENT '关联的打卡记录ID',
  `content` VARCHAR(500) DEFAULT NULL COMMENT '通知内容（评论内容）',
  `is_read` TINYINT(1) DEFAULT 0 COMMENT '是否已读：0-未读，1-已读',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `read_at` DATETIME DEFAULT NULL COMMENT '阅读时间',
  PRIMARY KEY (`notification_id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_is_read` (`is_read`),
  KEY `idx_created_at` (`created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='通知表';
```

**2.4 创建 NotificationController**

```java
@RestController
@RequestMapping("/api/notification")
public class NotificationController {
    
    @Autowired
    private NotificationService notificationService;
    
    /**
     * 获取未读通知数量
     */
    @GetMapping("/unread/count")
    public AjaxResult getUnreadCount() {
        Long userId = SecurityUtils.getUserId();
        int count = notificationService.getUnreadCount(userId);
        return AjaxResult.success(count);
    }
    
    /**
     * 获取未读通知列表
     */
    @GetMapping("/unread")
    public AjaxResult getUnreadNotifications() {
        Long userId = SecurityUtils.getUserId();
        List<Notification> list = notificationService.getUnreadNotifications(userId);
        return AjaxResult.success(list);
    }
    
    /**
     * 获取通知列表
     */
    @GetMapping("/list")
    public AjaxResult getNotifications(@RequestParam(defaultValue = "20") Integer limit) {
        Long userId = SecurityUtils.getUserId();
        List<Notification> list = notificationService.getNotifications(userId, limit);
        return AjaxResult.success(list);
    }
    
    /**
     * 标记通知为已读
     */
    @PostMapping("/read/{notificationId}")
    public AjaxResult markAsRead(@PathVariable Long notificationId) {
        notificationService.markAsRead(notificationId);
        return AjaxResult.success();
    }
    
    /**
     * 标记所有通知为已读
     */
    @PostMapping("/read/all")
    public AjaxResult markAllAsRead() {
        Long userId = SecurityUtils.getUserId();
        notificationService.markAllAsRead(userId);
        return AjaxResult.success();
    }
}
```

**3.2 在 app.js 中添加轮询**

```javascript
App({
  globalData: {
    unreadNotificationCount: 0
  },
  
  onLaunch() {
    // ... 现有代码 ...
    
    // 启动通知轮询
    this.startNotificationPolling();
  },
  
  onShow() {
    // 应用显示时刷新通知
    this.checkNotifications();
  },
  
  /**
   * 启动通知轮询（每30秒检查一次）
   */
  startNotificationPolling() {
    this.checkNotifications();
    
    setInterval(() => {
      this.checkNotifications();
    }, 30000); // 30秒
  },
  
  /**
   * 检查未读通知
   */
  checkNotifications() {
    const token = wx.getStorageSync('token');
    if (!token) {
      return;
    }
    
    api.notificationAPI.getUnreadCount().then(res => {
      if (res.code === 200) {
        const count = res.data || 0;
        const oldCount = this.globalData.unreadNotificationCount;
        
        this.globalData.unreadNotificationCount = count;
        
        // 如果有新通知，显示提示
        if (count > oldCount && count > 0) {
          this.showNotificationToast(count - oldCount);
        }
        
        // 更新 tabBar 徽标
        if (count > 0) {
          wx.setTabBarBadge({
            index: 3, // 假设通知在第4个tab
            text: count > 99 ? '99+' : count.toString()
          });
        } else {
          wx.removeTabBarBadge({
            index: 3
          });
        }
      }
    }).catch(err => {
      console.error('检查通知失败:', err);
    });
  },
  
  /**
   * 显示通知提示
   */
  showNotificationToast(newCount) {
    wx.showToast({
      title: `收到${newCount}条新通知`,
      icon: 'none',
      duration: 2000
    });
    
    // 震动提示
    wx.vibrateShort();
  }
});
```

**3.4 通知页面逻辑**

`pages/notifications/index.js`:
```javascript
const api = require('../../utils/api');
import Toast from '@vant/weapp/toast/toast';

Page({
  data: {
    notifications: [],
    loading: true
  },

  onLoad() {
    this.loadNotifications();
  },

  onShow() {
    // 每次显示时刷新
    this.loadNotifications();
  },

  /**
   * 加载通知列表
   */
  loadNotifications() {
    this.setData({ loading: true });
    
    api.notificationAPI.getNotifications(50).then(res => {
      if (res.code === 200) {
        this.setData({
          notifications: res.data || [],
          loading: false
        });
      }
    }).catch(err => {
      Toast.fail('加载失败');
      this.setData({ loading: false });
    });
  },

  /**
   * 点击通知
   */
  onNotificationClick(e) {
    const { id, recordId } = e.currentTarget.dataset;
    
    // 标记为已读
    api.notificationAPI.markAsRead(id).then(() => {
      // 更新本地状态
      const notifications = this.data.notifications.map(item => {
        if (item.notificationId === id) {
          return { ...item, isRead: true };
        }
        return item;
      });
      this.setData({ notifications });
      
      // 跳转到打卡记录详情
      // TODO: 实现打卡记录详情页面
      Toast.success('已标记为已读');
    });
  },

  /**
   * 标记全部已读
   */
  markAllAsRead() {
    api.notificationAPI.markAllAsRead().then(() => {
      Toast.success('已全部标记为已读');
      this.loadNotifications();
      
      // 更新全局未读数
      const app = getApp();
      app.globalData.unreadNotificationCount = 0;
      wx.removeTabBarBadge({ index: 3 });
    }).catch(err => {
      Toast.fail('操作失败');
    });
  }
});
```

### 方案2：WebSocket 实时推送（高级方案）

**优点**：
- 实时性强
- 用户体验好
- 减少轮询请求

**缺点**：
- 实现复杂

- 维护成本高

**实现要点**：
1. 后端使用 Spring WebSocket
2. 前端使用微信小程序的 WebSocket API
3. 建立长连接，实时推送通知

```javascript
function formatNotification(notification) {
  const messages = {
    like: `${notification.fromUser.nickname} 赞了你的打卡`,
    comment: `${notification.fromUser.nickname}: ${notification.content}`
  };
  return messages[notification.type] || '新通知';
}
```

### 3. 通知时间显示

```javascript
function formatNotificationTime(time) {
  const now = new Date();
  const notificationTime = new Date(time);
  const diff = now - notificationTime;
  
  if (diff < 60000) return '刚刚';
  if (diff < 3600000) return `${Math.floor(diff / 60000)}分钟前`;
  if (diff < 86400000) return `${Math.floor(diff / 3600000)}小时前`;
  return `${Math.floor(diff / 86400000)}天前`;
}
```

## 性能优化

### 1. 轮询频率控制
- 应用在前台：30秒轮询一次
- 应用在后台：停止轮询
- 用户打开通知页面：立即刷新

### 2. 缓存策略
- 缓存最近的通知列表
- 只请求增量数据
- 使用本地存储减少请求

### 3. 批量操作
- 批量标记已读
- 批量删除通知
- 减少数据库操作次数

## 总结

建议先实现方案1（数据库轮询），这是最简单实用的方案。等用户量增大后，再考虑升级到 WebSocket 实时推送。

关键点：
1. 在点赞/评论时创建通知记录
2. 前端定时轮询未读通知数量
3. 显示通知提示和 TabBar 徽标
4. 提供通知列表页面查看详情
