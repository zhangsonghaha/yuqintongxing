# 首页打卡记录不显示问题 - 调试步骤

## 问题描述
登录后首页不显示打卡记录，控制台有报错。

## 已完成的修复
1. ✅ 修复了 `app.js` 中的登录跳转逻辑冲突
2. ✅ 修复了首页 `loadData()` 方法的错误处理
3. ✅ 临时禁用了成就系统API调用（该功能尚未实现）
4. ✅ 增强了日志输出，便于排查问题

## 调试步骤

### 1. 清除缓存并重新编译
在微信开发者工具中：
- 点击"清缓存" → "清除全部缓存"
- 点击"编译" 重新编译项目

### 2. 重新登录
- 完全退出登录（如果已登录）
- 重新进入登录页面
- 点击"微信登录"按钮

### 3. 查看控制台日志
登录成功后，查看控制台输出，重点关注以下日志：

#### 关键日志标识
- `【首页】` - 首页相关日志
- `【网络请求】` - API请求日志

#### 需要检查的日志
```
【首页】用户信息: {...}
【首页】伴侣信息响应: {...}
【首页】用户统计数据响应: {...}
【首页】开始获取最近打卡记录...
【网络请求】发起请求: /api/checkin/recent?limit=3
【网络请求】请求成功: {...}
【首页】最近打卡记录原始响应: {...}
【首页】解析后的打卡记录数组: [...]
【首页】打卡记录数量: X
【首页】数据加载完成
```

### 4. 检查具体错误

#### 情况A：用户信息为空
如果看到：
```
【首页】用户信息不完整，跳转到登录页
```
**原因**：登录后用户信息未正确保存
**解决**：检查 `storage.js` 和登录逻辑

#### 情况B：API调用失败
如果看到：
```
【首页】获取最近打卡记录失败: Error: ...
```
**原因**：后端API返回错误
**解决**：
1. 检查后端服务是否正常运行
2. 检查数据库连接是否正常
3. 查看后端日志

#### 情况C：响应格式不匹配
如果看到：
```
【首页】未知的响应格式: {...}
```
**原因**：后端返回的数据格式与前端预期不符
**解决**：查看响应的具体格式，调整前端解析逻辑

#### 情况D：数据库无记录
如果看到：
```
【首页】打卡记录数量: 0
```
**原因**：数据库中没有打卡记录
**解决**：先完成一次打卡，然后刷新首页

### 5. 测试打卡功能
如果首页加载正常但没有记录：
1. 点击"打卡（3秒完成）"按钮
2. 填写打卡信息并提交
3. 返回首页查看是否显示

### 6. 检查数据库
如果前端一切正常但仍无数据，直接查询数据库：

```sql
-- 检查用户表
SELECT * FROM sys_user WHERE user_id = YOUR_USER_ID;

-- 检查打卡记录表
SELECT * FROM check_in_record WHERE user_id = YOUR_USER_ID AND deleted_at IS NULL;

-- 检查最近3条记录
SELECT * FROM check_in_record 
WHERE user_id = YOUR_USER_ID AND deleted_at IS NULL
ORDER BY check_in_date DESC, created_at DESC
LIMIT 3;
```

## 常见问题

### Q1: 控制台显示 "No message available"
**A**: 这是之前的错误，已通过修复 `request.js` 解决。如果仍出现，请确保已更新代码。

### Q2: 首页一直显示"加载中..."
**A**: 可能是某个API调用卡住了。查看控制台最后一条日志，确定卡在哪个API。

### Q3: 显示"加载失败"
**A**: 查看控制台的错误信息，根据错误类型进行排查。

### Q4: 后端返回401错误
**A**: Token过期或无效，需要重新登录。
