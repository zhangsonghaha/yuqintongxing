# 调试首页打卡状态显示问题

## 问题描述
登录后首页显示"我的状态"和"对方的状态"都显示"未打卡"，但实际上已经打过卡了。

## 排查步骤

### 1. 检查后端数据
在微信开发者工具的控制台中查看日志，搜索关键字：`【首页】用户统计数据响应`

应该能看到类似这样的日志：
```javascript
【首页】用户统计数据响应: {
  code: 200,
  data: {
    todayCheckedIn: true,  // 这个值应该是 true
    todayCheckInTime: "2026-03-12 10:30:00",
    weeklyCheckIns: 1,
    weeklyDuration: 123,
    // ... 其他字段
  }
}
```

**检查点**：
- `todayCheckedIn` 的值是 `true` 还是 `false`？
- 如果是 `false`，说明后端查询有问题
- 如果是 `true`，说明前端显示逻辑有问题

### 2. 检查数据库
在数据库中执行以下 SQL 查询，确认今天是否有打卡记录：

```sql
-- 查看今天的打卡记录
SELECT * FROM check_in_record 
WHERE user_id = YOUR_USER_ID 
AND DATE(check_in_date) = CURDATE()
ORDER BY created_at DESC;
```

**检查点**：
- 是否有今天的打卡记录？
- `check_in_date` 字段的值是否是今天的日期？
- `created_at` 字段的值是否是今天？

### 3. 检查后端查询逻辑
查看后端日志，搜索关键字：`今日打卡记录`

应该能看到类似这样的日志：
```
今日打卡记录: CheckInRecord{recordId=123, userId=1, checkInDate=2026-03-12, ...}
```

**检查点**：
- 后端是否查询到了今天的打卡记录？
- 如果没有查询到，可能是日期比较逻辑有问题

### 4. 检查前端数据绑定
在微信开发者工具的 AppData 面板中查看 `todayStatus` 的值：

```javascript
todayStatus: {
  userCheckedIn: false,  // 这个值应该是 true
  partnerCheckedIn: false,
  userCheckInTime: "",
  partnerCheckInTime: ""
}
```

**检查点**：
- `userCheckedIn` 的值是什么？
- 如果是 `false`，说明数据没有正确传递到页面

## 可能的原因

### 原因 1：日期时区问题
后端使用的日期可能与数据库中的日期不一致。

**解决方案**：
检查 `CheckInServiceImpl.java` 中的日期比较逻辑：
```java
// 查询今日打卡记录
CheckInRecord todayCheckIn = checkInMapper.selectTodayCheckIn(userId);
```

确保 `selectTodayCheckIn` 方法使用正确的日期比较：
```sql
WHERE user_id = #{userId}
AND DATE(check_in_date) = CURDATE()
```

### 原因 2：数据响应格式问题
前端可能没有正确解析后端返回的数据。

**解决方案**：
在 `couple-fitness-weapp/pages/index/index.js` 的第 115 行添加详细日志：
```javascript
const userStatsResponse = await checkInAPI.getStatistics();
console.log('【首页】用户统计数据原始响应:', JSON.stringify(userStatsResponse));
console.log('【首页】todayCheckedIn 值:', userStatsResponse.data.todayCheckedIn);
console.log('【首页】todayCheckedIn 类型:', typeof userStatsResponse.data.todayCheckedIn);
```

### 原因 3：数据缓存问题
可能是旧数据被缓存了。

**解决方案**：
1. 在微信开发者工具中点击"清除缓存" → "清除数据缓存"
2. 重新编译项目
3. 重新登录

### 原因 4：API 调用失败
API 调用可能失败了，但错误被忽略了。

**解决方案**：
检查控制台是否有错误日志：
```
【首页】获取用户统计数据失败: xxx
```

## 快速修复方案

如果以上排查都没有问题，可以尝试以下临时修复：

### 方案 1：强制刷新数据
在首页添加一个"刷新"按钮，手动触发数据加载：

```javascript
// 在 couple-fitness-weapp/pages/index/index.js 中添加
onRefresh() {
  console.log('【首页】手动刷新数据');
  this.loadData();
}
```

### 方案 2：添加下拉刷新
在 `couple-fitness-weapp/pages/index/index.json` 中启用下拉刷新：
```json
{
  "enablePullDownRefresh": true
}
```

然后在 `index.js` 中添加：
```javascript
onPullDownRefresh() {
  this.loadData().then(() => {
    wx.stopPullDownRefresh();
  });
}
```
