# 点赞和评论功能说明

## 功能概述

点赞和评论功能允许情侣之间对彼此的打卡记录进行互动，增强情感连接。

## 功能位置

### 1. 首页
- 在"最近打卡"区域
- 每条打卡记录下方显示互动栏
- 包含点赞和评论按钮

### 2. 日历页面
- 点击日历上的打卡日期
- 查看打卡详情
- 详情页面显示互动功能

## 使用方法

### 点赞功能

**操作步骤**：
1. 找到要点赞的打卡记录
2. 点击 ❤️ 图标
3. 图标变为红色，点赞数+1
4. 再次点击可取消点赞

**功能特点**：
- 每个用户对每条记录只能点赞一次
- 点赞后图标变红
- 实时更新点赞数量
- 支持取消点赞

### 评论功能

**操作步骤**：
1. 找到要评论的打卡记录
2. 点击 💬 图标
3. 在弹出的输入框中输入评论
4. 点击"发送"按钮
5. 评论显示在评论列表中

**功能特点**：
- 支持多条评论
- 显示评论者昵称和头像
- 显示评论时间
- 可以删除自己的评论
- 支持快捷回复（预设短语）

## API接口

### 点赞相关

**点赞**
```
POST /api/interaction/like
Body: {
  "recordId": 打卡记录ID,
  "userId": 用户ID
}
```

**取消点赞**
```
DELETE /api/interaction/like?recordId=xxx&userId=xxx
```

**检查是否已点赞**
```
GET /api/interaction/hasLiked?recordId=xxx&userId=xxx
```

### 评论相关

**发表评论**
```
POST /api/interaction/comment
Body: {
  "recordId": 打卡记录ID,
  "userId": 用户ID,
  "content": "评论内容"
}
```

**删除评论**
```
DELETE /api/interaction/comment/{interactionId}?userId=xxx
```

**获取互动列表**
```
GET /api/interaction/list/{recordId}
```

**获取统计数据**
```
GET /api/interaction/stats/{recordId}
```

## 数据库表结构

**interaction 表**
```sql
CREATE TABLE interaction (
  interaction_id BIGINT PRIMARY KEY AUTO_INCREMENT,
  record_id BIGINT NOT NULL COMMENT '打卡记录ID',
  user_id BIGINT NOT NULL COMMENT '用户ID',
  type VARCHAR(20) NOT NULL COMMENT '互动类型：like/comment',
  content TEXT COMMENT '评论内容',
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  deleted_at DATETIME DEFAULT NULL,
  INDEX idx_record_id (record_id),
  INDEX idx_user_id (user_id)
);
```

## 常见问题

### Q1: 点赞按钮点击后没反应
**A**: 
1. 检查后端服务是否正常
2. 查看控制台是否有错误
3. 确认用户已登录
4. 检查网络请求是否成功

### Q2: 评论提交后不显示
**A**: 
1. 检查评论内容是否为空
2. 查看后端日志是否有错误
3. 确认数据库连接正常
4. 刷新页面重新加载

### Q3: 无法删除评论
**A**: 
1. 确认是否是自己的评论
2. 检查用户权限
3. 查看后端日志

### Q4: 首页没有显示打卡记录
**A**: 
1. 先完成一次打卡
2. 返回首页刷新
3. 检查数据库是否有记录
4. 查看控制台日志
