# 目标管理表部署指南

## 问题描述

后端启动时报错：
```
Caused by: java.sql.SQLSyntaxErrorException: Table 'ruoyi-vue-pro.goal' doesn't exist
```

这是因为数据库中缺少 `goal` 表。

## 解决方案

### 方法 1：使用 MySQL 命令行（推荐）

1. **打开命令行，连接到 MySQL**：
```bash
mysql -u root -p
```

2. **选择数据库**：
```sql
USE ruoyi-vue-pro;
-- 或者你的数据库名称，例如：
-- USE couple_fitness;
```

3. **执行 SQL 脚本**：
```bash
source goal-table.sql;
-- 或者使用绝对路径：
-- source /path/to/goal-table.sql;
```

### 方法 2：使用 MySQL Workbench

1. 打开 MySQL Workbench
2. 连接到你的数据库
3. 打开 `goal-table.sql` 文件
4. 点击"执行"按钮（闪电图标）

### 方法 3：使用 Navicat

1. 打开 Navicat
2. 连接到你的数据库
3. 右键点击数据库 → "执行 SQL 文件"
4. 选择 `goal-table.sql` 文件
5. 点击"开始"

### 方法 4：直接复制 SQL 执行

复制以下 SQL 并在数据库客户端中执行：

```sql
-- 目标管理表
CREATE TABLE IF NOT EXISTS `goal` (
  `goal_id` BIGINT(20) NOT NULL AUTO_INCREMENT COMMENT '目标ID',
  `user_id` BIGINT(20) NOT NULL COMMENT '用户ID',
  `goal_type` VARCHAR(20) NOT NULL COMMENT '目标类型：weekly-周目标, monthly-月目标',
  `target_value` INT(11) NOT NULL COMMENT '目标值（周目标：次数，月目标：分钟数）',
  `current_value` INT(11) DEFAULT 0 COMMENT '当前完成值',
  `start_date` DATE NOT NULL COMMENT '目标开始日期',
  `end_date` DATE NOT NULL COMMENT '目标结束日期',
  `status` VARCHAR(20) DEFAULT 'active' COMMENT '目标状态：active-进行中, completed-已完成, expired-已过期',
  `completed_at` DATETIME DEFAULT NULL COMMENT '完成时间',
  `created_at` DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  `deleted_at` DATETIME DEFAULT NULL COMMENT '删除时间（软删除）',
  PRIMARY KEY (`goal_id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_goal_type` (`goal_type`),
  KEY `idx_status` (`status`),
  KEY `idx_start_date` (`start_date`),
  KEY `idx_end_date` (`end_date`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='目标管理表';
```

## 常见问题

### Q1: 数据库名称不对怎么办？

**A**: 检查你的数据库名称，可能是：
- `ruoyi-vue-pro`
- `couple_fitness`
- `ry-vue`

在执行 SQL 前，先使用 `USE database_name;` 切换到正确的数据库。

### Q2: 权限不足怎么办？

**A**: 确保你的 MySQL 用户有 `CREATE TABLE` 权限：
```sql
GRANT CREATE ON database_name.* TO 'your_user'@'localhost';
FLUSH PRIVILEGES;
```

### Q3: 表已存在怎么办？

**A**: SQL 脚本使用了 `CREATE TABLE IF NOT EXISTS`，所以不会报错。如果需要重新创建，先删除表：
```sql
DROP TABLE IF EXISTS goal;
```
然后重新执行创建脚本。

### Q4: 字符集问题怎么办？

**A**: 确保数据库和表使用 UTF-8 字符集：
```sql
ALTER DATABASE database_name CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```
