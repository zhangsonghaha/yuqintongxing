# 腾讯地图位置打卡功能部署指南

## 功能概述
在打卡时自动获取用户当前位置，使用腾讯地图服务进行逆地址解析，记录打卡的具体位置信息。

## 部署步骤

### 1. 数据库更新
```sql
-- 方法1：使用迁移脚本（推荐）
mysql -u root -p your_database < RuoYi-Vue/sql/add_location_fields_to_checkin.sql

-- 方法2：手动执行SQL
ALTER TABLE check_in_record 
ADD COLUMN latitude DECIMAL(10,6) COMMENT '纬度',
ADD COLUMN longitude DECIMAL(10,6) COMMENT '经度',
ADD COLUMN location VARCHAR(200) COMMENT '位置描述';
```

### 2. 后端服务重启
```bash
# 进入后端项目目录
cd RuoYi-Vue/ruoyi-admin

# 清理并重新编译
mvn clean compile

# 启动服务
mvn spring-boot:run
```

## 故障排除

### 问题1：位置获取失败
**症状**：点击"获取位置"按钮后无反应或显示"位置获取失败"
**解决方案**：
1. 检查微信开发者工具中的位置模拟设置
2. 确认小程序已配置位置权限
3. 检查网络连接
4. 查看控制台错误信息

### 问题2：地址解析失败
**症状**：获取到坐标但无法显示地址
**解决方案**：
1. 检查腾讯地图API Key是否正确
2. 确认API Key有足够的调用额度
3. 检查网络连接
4. 查看腾讯地图API返回的错误信息

### 问题3：数据存储失败
**症状**：打卡成功但位置数据未保存
**解决方案**：
1. 检查数据库表结构是否正确
2. 查看后端日志中的SQL错误
3. 确认MyBatis Mapper配置正确
4. 检查实体类字段类型是否匹配

## 回滚方案

如果新功能出现问题，可按以下步骤回滚：

### 步骤1：前端回滚
1. 恢复 `couple-fitness-weapp/utils/location.js` 到之前版本
2. 移除打卡页面中的位置相关代码

### 步骤2：后端回滚
1. 恢复 `CheckInRecord` 实体类到之前版本
2. 恢复 `CheckInMapper.xml` 到之前版本
3. 重新编译并启动后端服务

### 步骤3：数据库回滚
```sql
-- 移除位置字段（谨慎操作，会丢失位置数据）
ALTER TABLE check_in_record 
DROP COLUMN latitude,
DROP COLUMN longitude,
DROP COLUMN location;
```
