# 修复日期筛选参数传递问题

## 问题描述

点击日历上的 3月11日，后端执行的 SQL 没有带上日期筛选条件：

```sql
SELECT * FROM check_in_record
WHERE deleted_at IS NULL
AND user_id = 37
ORDER BY check_in_date DESC, created_at DESC
```

后端接收到的参数全部是 `null`：
- `beginTime = null`
- `endTime = null`
- `params` Map 中所有参数都是 `null`

## 根本原因

**Spring MVC 不会自动将 URL 参数放入 `BaseEntity` 的 `params` Map 中**。

RuoYi 框架的 `BaseEntity` 提供了一个 `params` 属性（Map 类型），但 Spring MVC 默认只会绑定对象的直接属性（如 `userId`、`checkInDate`），不会自动填充 `params` Map。

**错误的假设**：
```java
// ❌ 错误：以为 Spring MVC 会自动将 beginTime 和 endTime 放入 params
@GetMapping("/list")
public TableDataInfo listCheckIn(CheckInRecord checkInRecord) {
    // checkInRecord.getParams() 是空的！
}
```

## 解决方案

### 方案 1：在 Controller 中手动设置 params（推荐）✅

在 Controller 方法中显式接收 `beginTime` 和 `endTime` 参数，然后手动放入 `params` Map：

```java
@GetMapping("/list")
public TableDataInfo listCheckIn(CheckInRecord checkInRecord,
                                  @RequestParam(required = false) String beginTime,
                                  @RequestParam(required = false) String endTime) {
    // 设置当前用户ID
    Long userId = SecurityUtils.getUserId();
    checkInRecord.setUserId(userId);
    
    // ✅ 手动设置 params 参数
    if (beginTime != null && !beginTime.isEmpty()) {
        checkInRecord.getParams().put("beginTime", beginTime);
    }
    if (endTime != null && !endTime.isEmpty()) {
        checkInRecord.getParams().put("endTime", endTime);
    }
    
    // 打印调试信息
    log.info("【打卡记录列表】userId: {}", userId);
    log.info("【打卡记录列表】beginTime: {}", beginTime);
    log.info("【打卡记录列表】endTime: {}", endTime);
    log.info("【打卡记录列表】params: {}", checkInRecord.getParams());
    
    startPage();
    List<CheckInRecord> list = checkInService.getCheckInList(checkInRecord);
    return getDataTable(list);
}
```

**优点**：
- 简单直接，易于理解
- 不需要修改框架代码
- 可以对参数进行验证和处理

### 方案 2：使用 @RequestParam Map（备选）

```java
@GetMapping("/list")
public TableDataInfo listCheckIn(CheckInRecord checkInRecord,
                                  @RequestParam Map<String, String> params) {
    // 设置当前用户ID
    Long userId = SecurityUtils.getUserId();
    checkInRecord.setUserId(userId);
    
    // 将所有额外参数放入 params
    checkInRecord.getParams().putAll(params);
    
    startPage();
    List<CheckInRecord> list = checkInService.getCheckInList(checkInRecord);
    return getDataTable(list);
}
```

**缺点**：
- 会将所有 URL 参数都放入 params，包括 `pageNum`、`pageSize` 等
- 可能导致意外的参数污染

```

```

### 3. 后端 Controller 接收

```java
@GetMapping("/list")
public TableDataInfo listCheckIn(
    CheckInRecord checkInRecord,           // Spring MVC 自动绑定
    @RequestParam(required = false) String beginTime,  // 显式接收
    @RequestParam(required = false) String endTime     // 显式接收
) {
    // 手动放入 params
    checkInRecord.getParams().put("beginTime", beginTime);
    checkInRecord.getParams().put("endTime", endTime);
}
```

### 4. MyBatis 使用参数

```xml

```

## 为什么之前没有发现这个问题？

1. **RuoYi 官方示例通常使用 `@RequestParam` 显式接收参数**
2. **我们误以为 Spring MVC 会自动填充 `params` Map**
3. **前端传递的参数名和后端期望的参数名一致，但没有正确传递到 MyBatis**

## 总结

**问题根源**：Spring MVC 不会自动将 URL 参数放入 `BaseEntity.params` Map

**解决方案**：在 Controller 中使用 `@RequestParam` 显式接收参数，然后手动放入 `params` Map

**修改的文件**：
1. ✅ `RuoYi-Vue/ruoyi-admin/src/main/java/com/ruoyi/web/controller/CheckInController.java`

**不需要修改的文件**：
- ❌ `CheckInMapper.xml` - SQL 查询逻辑是正确的
- ❌ `CheckInRecord.java` - 实体类定义是正确的
- ❌ 前端代码 - 参数传递是正确的

现在重启后端服务，日期筛选功能应该可以正常工作了！
