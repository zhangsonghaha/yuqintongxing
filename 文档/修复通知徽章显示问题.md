# 修复通知徽章显示问题

## 问题描述

在个人中心页面引用 `van-badge` 组件时报错：
```
component not found in the path: @vant/weapp/badge/index
```

## 问题原因

Vant Weapp 组件库中没有独立的 `badge` 组件。Vant 的徽章功能是通过 `info` 组件实现的，但 `info` 组件主要用于配合其他组件使用，不适合单独使用。

## 解决方案

使用纯 CSS 实现徽章效果，不依赖 Vant 组件。

### 修改内容

#### 1. 移除 Vant Badge 组件引用

**文件**: `couple-fitness-weapp/pages/profile/profile.json`

**修改前**:
```json
{
  "navigationBarTitleText": "我的",
  "usingComponents": {
    "van-badge": "@vant/weapp/badge/index"
  }
}
```

**修改后**:
```json
{
  "navigationBarTitleText": "我的"
}
```

#### 2. 使用自定义徽章元素

**文件**: `couple-fitness-weapp/pages/profile/profile.wxml`

**修改前**:
```xml
<view class="menu-item-left">
  <text class="menu-icon">🔔</text>
  <text class="menu-text">通知中心</text>
  <van-badge wx:if="{{ unreadCount > 0 }}" content="{{ unreadCount }}" max="99" />
</view>
```

**修改后**:
```xml
<view class="menu-item-left">
  <text class="menu-icon">🔔</text>
  <text class="menu-text">通知中心</text>
  <view wx:if="{{ unreadCount > 0 }}" class="unread-badge">
    {{ unreadCount > 99 ? '99+' : unreadCount }}
  </view>
</view>
```

#### 3. 添加徽章样式

**文件**: `couple-fitness-weapp/pages/profile/profile.wxss`

**添加的样式**:
```css
/* 未读徽章 */
.unread-badge {
  position: absolute;
  left: 28px;
  top: -4px;
  min-width: 16px;
  height: 16px;
  padding: 0 4px;
  background: linear-gradient(135deg, #FF6B9D 0%, #FF4081 100%);
  color: white;
  font-size: 10px;
  font-weight: bold;
  border-radius: 8px;
  display: flex;
  align-items: center;
  justify-content: center;
  box-shadow: 0 2px 4px rgba(255, 107, 157, 0.3);
  z-index: 10;
}
```

## 实现效果

### 视觉效果

- 红色渐变背景（粉色主题）
- 白色文字，粗体
- 圆角矩形
- 轻微阴影效果
- 定位在图标右上角

### 功能特点

1. **自动显示/隐藏**
   - 未读数 > 0 时显示
   - 未读数 = 0 时隐藏

2. **数字格式化**
   - 未读数 ≤ 99：显示具体数字
   - 未读数 > 99：显示 "99+"

3. **响应式尺寸**
   - 最小宽度 16px
   - 根据数字长度自动扩展
   - 高度固定 16px

## 优势

### 相比 Vant 组件

1. **无依赖**
   - 不依赖外部组件
   - 减少组件引用
   - 避免版本兼容问题

2. **完全可控**
   - 样式完全自定义
   - 可以精确控制位置
   - 易于调整和维护

3. **性能更好**
   - 纯 CSS 实现
   - 无 JS 逻辑开销
   - 渲染性能更优

4. **符合设计**
   - 使用项目主色调
   - 与整体风格一致
   - 视觉效果更好

## 使用示例

### 基本使用

```xml
<view class="menu-item-left">
  <text class="menu-icon">🔔</text>
  <text class="menu-text">通知中心</text>
  <view wx:if="{{ unreadCount > 0 }}" class="unread-badge">
    {{ unreadCount > 99 ? '99+' : unreadCount }}
  </view>
</view>
```

### 在其他位置使用

如果需要在其他地方使用徽章，只需：

1. 复制 `.unread-badge` 样式到对应的 wxss 文件
2. 调整 `left` 和 `top` 属性来定位徽章
3. 在 wxml 中使用相同的结构

**示例**（在头像上显示徽章）:
```xml
<view style="position: relative;">
  <image class="avatar" src="{{ avatar }}" />
  <view wx:if="{{ messageCount > 0 }}" class="unread-badge" style="left: 40px; top: 0;">
    {{ messageCount }}
  </view>
</view>
```

## 样式定制

### 修改颜色

```css
.unread-badge {
  /* 修改背景色 */
  background: linear-gradient(135deg, #4CAF50 0%, #45A049 100%); /* 绿色 */
  /* 或纯色 */
  background: #FF6B9D;
}
```

### 修改大小

```css
.unread-badge {
  min-width: 18px;  /* 最小宽度 */
  height: 18px;     /* 高度 */
  font-size: 11px;  /* 字体大小 */
}
```

### 修改位置

```css
.unread-badge {
  left: 30px;   /* 水平位置 */
  top: -6px;    /* 垂直位置 */
}
```

### 修改形状

```css
.unread-badge {
  border-radius: 50%;  /* 圆形 */
  /* 或 */
  border-radius: 4px;  /* 小圆角 */
}
```

通过使用纯 CSS 实现徽章效果，成功解决了 Vant Badge 组件不存在的问题。这个方案不仅解决了问题，还带来了更好的性能和更灵活的定制能力。

- 使用 `position: absolute` 定位徽章
- 使用渐变背景符合主题风格

- 完全响应式，自动适应数字长度
