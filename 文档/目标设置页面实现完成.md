### 4. API 集成

#### 4.1 新增 API 函数

在 `utils/api.js` 中添加了 `goalAPI` 对象：

```javascript
var goalAPI = {
  // 获取当前活跃目标
  getActiveGoal: function(goalType) {
    return request.get('/api/goal/active?goalType=' + goalType);
  },
  
  // 创建目标
  createGoal: function(goalType, targetValue) {
    return request.post('/api/goal/create', {
      goalType: goalType,
      targetValue: targetValue
    });
  },
  
  // 更新目标
  updateGoal: function(goalId, targetValue) {
    return request.put('/api/goal/update', {
      goalId: goalId,
      targetValue: targetValue
    });
  },
  
  // 检查目标完成情况
  checkGoalProgress: function(goalType) {
    return request.get('/api/goal/check?goalType=' + goalType);
  }
};
```

#### 4.2 后端 API 端点

对接以下后端 API：

- `GET /api/goal/active?goalType=weekly` - 获取周目标
- `GET /api/goal/active?goalType=monthly` - 获取月目标
- `POST /api/goal/create` - 创建目标
- `PUT /api/goal/update` - 更新目标
- `GET /api/goal/check?goalType=weekly` - 检查周目标进度
- `GET /api/goal/check?goalType=monthly` - 检查月目标进度

## 使用方法

### 访问页面

```javascript
// 从其他页面跳转到目标设置页面
wx.navigateTo({
  url: '/pages/goal/index'
});
```

### 页面功能

1. **设置周目标**
   - 在"每周打卡次数"输入框输入 1-7 的数字
   - 点击"保存目标"按钮

2. **设置月目标**
   - 在"每月运动时长"输入框输入 1-3000 的数字
   - 点击"保存目标"按钮

3. **查看进度**
   - 页面自动显示当前目标完成进度
   - 进度条实时更新

4. **修改目标**
   - 修改输入框中的数值
   - 点击"保存目标"按钮更新

## 数据流程

### 加载流程

```
页面加载
  ↓
调用 loadGoals()
  ↓
并行加载周目标和月目标
  ↓
loadWeeklyGoal() + loadMonthlyGoal()
  ↓
如果有目标，调用 checkProgress()
  ↓
显示目标值和进度
```

### 保存流程

```
用户输入目标值
  ↓
点击保存按钮
  ↓
验证输入（范围检查）
  ↓
判断是创建还是更新
  ↓
调用相应 API
  ↓
显示成功提示
  ↓
重新加载目标数据
```

## 验证规则

### 周目标
- 必须是数字
- 范围：1-7 次
- 错误提示："周目标请输入1-7次"

### 月目标
- 必须是数字
- 范围：1-3000 分钟
- 错误提示："月目标请输入1-3000分钟"

### 通用验证
- 至少设置一个目标
- 错误提示："请至少设置一个目标"

## 错误处理

### 网络错误
- 加载失败：console.error 记录
- 保存失败：Toast 提示 "保存失败，请重试"

### 验证错误
- 输入范围错误：Toast 提示具体错误信息
- 未设置目标：Toast 提示 "请至少设置一个目标"

### 权限错误
- 未登录：后端返回 "用户未登录"
- 无权限：后端返回相应错误信息
