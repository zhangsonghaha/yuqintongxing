# 修复点赞状态丢失问题

## 问题描述

用户在"对方打卡记录"页面点赞后，返回首页发现点赞状态丢失了。

## 问题原因

1. 点赞操作只更新了当前页面的本地数据
2. 没有在页面返回时通知首页刷新数据
3. 首页的打卡记录列表没有实时同步

## 解决方案

### 方案1：页面卸载时通知首页刷新（已实现）

在对方打卡记录页面卸载时，如果数据有更新，通知首页进行静默刷新。

**修改文件**: `couple-fitness-weapp/pages/partner-checkins/index.js`

#### 1. 添加数据变更标记

```javascript
Page({
  data: {
    // ...
  },
  
  // 标记数据是否已更新
  dataChanged: false,
  
  // ...
});
```

#### 2. 点赞时设置标记

```javascript
onLikeChange(e) {
  const { recordId, isLiked, likeCount } = e.detail;
  
  // 更新本地数据
  const checkInList = this.data.checkInList.map(item => {
    if (item.recordId === recordId) {
      return {
        ...item,
        hasLiked: isLiked,
        likeCount: likeCount
      };
    }
    return item;
  });
  
  this.setData({ checkInList });
  
  // 标记数据已更新，返回时需要刷新首页
  this.dataChanged = true;
}
```

#### 3. 页面卸载时通知首页

```javascript
/**
 * 页面卸载时，如果数据有更新，通知首页刷新
 */
onUnload() {
  if (this.dataChanged) {
    // 获取首页实例
    const pages = getCurrentPages();
    if (pages.length > 1) {
      const prevPage = pages[pages.length - 2];
      // 如果上一页是首页，触发刷新
      if (prevPage.route === 'pages/index/index') {
        prevPage.silentRefresh && prevPage.silentRefresh();
      }
    }
  }
}
```

### 方案2：使用 EventChannel（更优雅）

微信小程序提供了 EventChannel 用于页面间通信。

#### 首页跳转时设置监听

```javascript
goToPartnerCheckIns() {
  const { partnerInfo } = this.data;
  
  // ...
  
  wx.navigateTo({
    url: `/pages/partner-checkins/index?partnerId=${partnerId}&partnerName=${encodeURIComponent(partnerName)}`,
    events: {
      // 监听数据更新事件
      dataChanged: (data) => {
        console.log('【首页】收到数据更新通知:', data);
        this.silentRefresh();
      }
    },
    success: (res) => {
      // 可以向被打开页面传送数据
      res.eventChannel.emit('acceptDataFromOpenerPage', { data: 'test' });
    }
  });
}
```

#### 对方打卡记录页面发送事件

```javascript
onLikeChange(e) {
  const { recordId, isLiked, likeCount } = e.detail;
  
  // 更新本地数据
  // ...
  
  // 通知上一页数据已更新
  const eventChannel = this.getOpenerEventChannel();
  if (eventChannel) {
    eventChannel.emit('dataChanged', {
      recordId,
      isLiked,
      likeCount,
      timestamp: Date.now()
    });
  }
}
```

### 方案3：使用全局状态管理

在 app.js 中维护全局状态。

#### app.js

```javascript
App({
  globalData: {
    // 打卡记录缓存
    checkInListCache: {},
    // 最后更新时间
    lastUpdateTime: null
  },
  
  /**
   * 更新打卡记录缓存
   */
  updateCheckInCache(recordId, data) {
    this.globalData.checkInListCache[recordId] = {
      ...this.globalData.checkInListCache[recordId],
      ...data,
      updatedAt: Date.now()
    };
    this.globalData.lastUpdateTime = Date.now();
  },
  
  /**
   * 获取打卡记录缓存
   */
  getCheckInCache(recordId) {
    return this.globalData.checkInListCache[recordId];
  },
  
  /**
   * 清空缓存
   */
  clearCheckInCache() {
    this.globalData.checkInListCache = {};
    this.globalData.lastUpdateTime = null;
  }
});
```

#### 使用缓存

```javascript
// 点赞时更新缓存
onLikeChange(e) {
  const { recordId, isLiked, likeCount } = e.detail;
  
  // 更新本地数据
  // ...
  
  // 更新全局缓存
  const app = getApp();
  app.updateCheckInCache(recordId, {
    hasLiked: isLiked,
    likeCount: likeCount
  });
}

// 首页加载时检查缓存
onShow() {
  const app = getApp();
  const lastUpdateTime = app.globalData.lastUpdateTime;
  
  // 如果有更新，刷新数据
  if (lastUpdateTime && lastUpdateTime > this.lastLoadTime) {
    this.silentRefresh();
  }
}
```

## 推荐方案

**推荐使用方案1（页面卸载通知）**，原因：

1. ✅ 实现简单，代码量少
2. ✅ 不需要修改跳转逻辑
3. ✅ 性能开销小
4. ✅ 兼容性好

**方案2（EventChannel）** 更优雅，但需要修改跳转逻辑。

**方案3（全局状态）** 适合复杂场景，但对于当前需求过于复杂。

### 3. 优化刷新策略

只刷新有变化的记录，而不是整个列表：

```javascript
silentRefresh() {
  // 只更新有变化的记录
  const app = getApp();
  const cache = app.globalData.checkInListCache;
  
  const recentCheckIns = this.data.recentCheckIns.map(item => {
    const cached = cache[item.recordId];
    if (cached && cached.updatedAt > this.lastLoadTime) {
      return { ...item, ...cached };
    }
    return item;
  });
  
  this.setData({ recentCheckIns });
}
```

### 4. 添加防抖

避免频繁刷新：

```javascript
// 防抖函数
debounce(fn, delay = 300) {
  let timer = null;
  return function(...args) {
    if (timer) clearTimeout(timer);
    timer = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}

// 使用防抖
this.debouncedRefresh = this.debounce(this.silentRefresh, 500);

onUnload() {
  if (this.dataChanged) {
    // 使用防抖刷新
    const pages = getCurrentPages();
    if (pages.length > 1) {
      const prevPage = pages[pages.length - 2];
      if (prevPage.route === 'pages/index/index') {
        prevPage.debouncedRefresh && prevPage.debouncedRefresh();
      }
    }
  }
}
```
