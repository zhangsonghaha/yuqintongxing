# 解决方案：创建数据库表和显示双方打卡记录

## 问题1：数据库表不存在

### 错误信息
```
Table 'ruoyi-vue-pro.interaction' doesn't exist
```

### 解决方案：创建 interaction 表

**步骤1：连接数据库**
```bash
mysql -u root -p
```

**步骤2：选择数据库**
```sql
USE `ruoyi-vue-pro`;  -- 或者你的数据库名
```

**步骤3：创建表**
```sql
CREATE TABLE IF NOT EXISTS `interaction` (
  `interaction_id` BIGINT(20) NOT NULL AUTO_INCREMENT COMMENT '互动ID',
  `record_id` BIGINT(20) NOT NULL COMMENT '打卡记录ID',
  `user_id` BIGINT(20) NOT NULL COMMENT '用户ID',
  `type` VARCHAR(20) NOT NULL COMMENT '互动类型：like-点赞, comment-评论',
  `content` TEXT COMMENT '评论内容（点赞时为空）',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `deleted_at` DATETIME DEFAULT NULL COMMENT '删除时间（软删除）',
  PRIMARY KEY (`interaction_id`),
  INDEX `idx_record_id` (`record_id`),
  INDEX `idx_user_id` (`user_id`),
  INDEX `idx_type` (`type`),
  INDEX `idx_created_at` (`created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='互动表（点赞和评论）';
```

**步骤4：验证表创建**
```sql
SHOW TABLES LIKE 'interaction';
DESC interaction;
```

## 问题2：只显示自己的打卡记录

### 问题分析
您说得对！情侣健身应用的核心是**双方互动**，应该显示：
- 自己的打卡记录
- 伴侣的打卡记录
- 可以互相点赞和评论

### 当前问题
- 首页只显示当前用户自己的打卡记录
- 无法看到伴侣的打卡
- 自己给自己点赞评论没有意义

### 解决方案

#### 方案A：修改现有API（推荐）

**后端修改**：在 `CheckInController` 添加新接口

```java
/**
 * 获取情侣双方的最近打卡记录
 */
@GetMapping("/recent/couple")
public AjaxResult getCoupleRecentCheckIns(@RequestParam(defaultValue = "10") Integer limit) {
    Long userId = SecurityUtils.getUserId();
    List<CheckInRecord> list = checkInService.getCoupleRecentCheckIns(userId, limit);
    return AjaxResult.success(list);
}
```

**Service实现**：
```java
@Override
public List<CheckInRecord> getCoupleRecentCheckIns(Long userId, Integer limit) {
    // 1. 获取用户的伴侣ID
    Long partnerId = partnershipService.getPartnerId(userId);
    
    // 2. 获取双方的打卡记录
    List<CheckInRecord> records = new ArrayList<>();
    
    // 用户自己的记录
    records.addAll(checkInMapper.selectRecentCheckIns(userId, limit / 2));
    
    // 伴侣的记录
    if (partnerId != null) {
        records.addAll(checkInMapper.selectRecentCheckIns(partnerId, limit / 2));
    }
    
    // 3. 按时间倒序排序
    records.sort((a, b) -> b.getCreatedAt().compareTo(a.getCreatedAt()));
    
    // 4. 限制数量
    return records.stream().limit(limit).collect(Collectors.toList());
}
```

## 数据库查询验证

### 查看互动记录
```sql
SELECT 
  i.interaction_id,
  i.record_id,
  i.user_id,
  u.nickname AS user_nickname,
  i.type,
  i.content,
  i.created_at
FROM interaction i
LEFT JOIN couple_user u ON i.user_id = u.user_id
WHERE i.deleted_at IS NULL
ORDER BY i.created_at DESC;
```

### 查看某条打卡的互动统计
```sql
SELECT 
  record_id,
  COUNT(CASE WHEN type = 'like' THEN 1 END) AS like_count,
  COUNT(CASE WHEN type = 'comment' THEN 1 END) AS comment_count
FROM interaction
WHERE record_id = ? AND deleted_at IS NULL
GROUP BY record_id;
```
