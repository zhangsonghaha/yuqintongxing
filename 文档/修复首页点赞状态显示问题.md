# 修复首页点赞状态显示问题

## 问题描述

在首页查看自己的打卡记录时，即使对方已经给我点过赞，爱心图标仍然显示灰色（未点赞状态），而不是红色（已点赞状态）。

## 问题原因

在 `CheckInServiceImpl.getRecentCheckIns()` 方法中，调用 `enrichInteractionData()` 填充互动数据时，传入的 `currentUserId` 参数错误。

**错误的逻辑**:
```java
public List<CheckInRecord> getRecentCheckIns(Long userId, Integer limit) {
    List<CheckInRecord> records = checkInMapper.selectRecentCheckIns(userId, limit);
    
    // 错误：传入的是打卡记录所有者的ID
    for (CheckInRecord record : records) {
        enrichInteractionData(record, userId);  // ❌ userId 是打卡记录所有者
    }
    
    return records;
}
```

**问题分析**:
- `userId` 参数是打卡记录的所有者（我自己）
- `enrichInteractionData` 方法会检查 `currentUserId` 是否给这条记录点过赞
- 但传入的是我自己的ID，所以检查的是"我是否给我自己点过赞"
- 实际上应该检查的是"当前登录用户（可能是对方）是否给我点过赞"

## 解决方案

修改 `getRecentCheckIns()` 方法，从 Spring Security 上下文中获取当前登录用户的ID，而不是使用打卡记录所有者的ID。

### 修改内容

**文件**: `RuoYi-Vue/ruoyi-admin/src/main/java/com/ruoyi/web/service/impl/CheckInServiceImpl.java`

**修改后的代码**:
```java
@Override
public List<CheckInRecord> getRecentCheckIns(Long userId, Integer limit) {
    if (userId == null) {
        throw new ServiceException("用户ID不能为空");
    }
    if (limit == null || limit <= 0) {
        limit = 10; // 默认查询10条
    }
    List<CheckInRecord> records = checkInMapper.selectRecentCheckIns(userId, limit);
    
    // 获取当前登录用户ID（用于检查是否已点赞）
    Long currentUserId = null;
    try {
        currentUserId = SecurityUtils.getUserId();  // ✅ 从 Security 上下文获取
    } catch (Exception e) {
        // 如果获取失败，使用传入的userId
        currentUserId = userId;
    }
    
    // 填充点赞和评论统计数据
    for (CheckInRecord record : records) {
        enrichInteractionData(record, currentUserId);  // ✅ 传入当前登录用户ID
    }
    
    return records;
}
```

## 工作原理

### 场景1：查看自己的打卡记录

```
用户A登录 → 查看首页（自己的打卡记录）
    ↓
调用 getRecentCheckIns(userId=A)
    ↓
从 Security 上下文获取 currentUserId = A
    ↓
检查：用户A是否给这条记录点过赞
    ↓
结果：hasLiked = false（自己不能给自己点赞）
    ↓
前端显示：灰色爱心
```

### 场景2：对方查看我的打卡记录

```
用户B登录 → 查看用户A的打卡记录
    ↓
调用 getRecentCheckIns(userId=A)
    ↓
从 Security 上下文获取 currentUserId = B  ← 关键！
    ↓
检查：用户B是否给这条记录点过赞
    ↓
结果：hasLiked = true（用户B之前点过赞）
    ↓
前端显示：红色爱心 ✅
```

### 场景3：在首页看到对方给我点过赞的记录

```
用户A登录 → 查看首页（自己的打卡记录）
    ↓
调用 getRecentCheckIns(userId=A)
    ↓
从 Security 上下文获取 currentUserId = A
    ↓
检查：用户A是否给这条记录点过赞
    ↓
结果：hasLiked = false
    ↓
但是 likeCount = 1（用户B点过赞）
    ↓
前端显示：灰色爱心 + 点赞数1
```

## 注意事项

### 首页的特殊情况

首页显示的是"我自己的打卡记录"，当前登录用户就是打卡记录的所有者，所以：
- `hasLiked` 永远是 `false`（自己不能给自己点赞）
- 但 `likeCount` 会显示对方的点赞数

这是正确的行为！因为：
1. 自己不能给自己点赞
2. 爱心显示灰色是正确的
3. 点赞数显示对方的点赞

### 对方打卡记录页面

在"对方打卡记录"页面：
- 查看的是对方的打卡记录
- 当前登录用户是我自己
- `hasLiked` 会正确显示我是否给对方点过赞
- 如果我点过赞，爱心显示红色 ✅

## 相关方法

### enrichInteractionData 方法

```java
private void enrichInteractionData(CheckInRecord record, Long currentUserId) {
    if (record == null || record.getRecordId() == null) {
        return;
    }
    
    // 统计点赞数
    int likeCount = interactionMapper.countLikesByRecordId(record.getRecordId());
    record.setLikeCount(likeCount);
    
    // 统计评论数
    int commentCount = interactionMapper.countCommentsByRecordId(record.getRecordId());
    record.setCommentCount(commentCount);
    
    // 检查当前用户是否已点赞
    if (currentUserId != null) {
        Interaction like = interactionMapper.selectLikeByRecordIdAndUserId(
            record.getRecordId(), currentUserId);
        record.setHasLiked(like != null);  // ← 关键：检查 currentUserId 是否点过赞
    } else {
        record.setHasLiked(false);
    }
}
```

通过修改 `getRecentCheckIns()` 方法，从 Spring Security 上下文中获取当前登录用户的ID，而不是使用打卡记录所有者的ID，成功修复了点赞状态显示问题。

**关键点**:
- 区分"打卡记录所有者"和"当前登录用户"
- 使用 `SecurityUtils.getUserId()` 获取当前登录用户
- `hasLiked` 字段表示"当前登录用户是否点过赞"
- 首页显示自己的记录时，爱心应该是灰色（自己不能给自己点赞）
- 查看对方记录时，如果点过赞，爱心应该是红色
