# 修复目标进度显示问题

## 问题描述

用户打卡后，目标设置页面的进度没有更新，始终显示 0/1次（0%）。

## 问题原因

通过MCP MySQL工具查询数据库发现：

1. **打卡记录正确**：用户今天有1次打卡记录（check_in_date = 2026-03-12）
2. **SQL查询正确**：本周打卡次数查询返回正确结果（1次）
3. **字段名不匹配**：
   - 前端代码使用 `res.data.currentValue`
   - 后端返回的字段是 `res.data.actualValue`

## 数据库查询结果

```sql
-- 打卡记录
check_in_id | user_id | exercise_type | duration | check_in_date | created_at
2           | 39      | 跑步          | 123123   | 2026-03-12    | 2026-03-12 18:35:16
1           | 37      | 跑步          | 213      | 2026-03-12    | 2026-03-12 18:31:24

-- 本周打卡次数
user_id | weekly_checkins
37      | 1
39      | 1

-- 活跃目标
goal_id | user_id | goal_type | target_value | current_value | status
1       | 37      | weekly    | 1            | 0             | active
2       | 37      | monthly   | 123          | 0             | active
```

## 解决方案

### 1. 修复字段名（已完成）

修改 `couple-fitness-weapp/pages/goal/index.js`：

```javascript
// 修改前
const current = res.data.currentValue || 0;

// 修改后
const current = res.data.actualValue || 0;
```

### 2. 添加页面自动刷新（已完成）

- 添加 `onShow` 生命周期函数，每次进入页面时刷新进度
- 添加 `refreshProgress` 方法，静默刷新进度数据
- 在 `loadGoals` 完成后自动刷新进度

### 3. 添加调试日志（已完成）

在进度检查方法中添加详细的日志输出，方便后续调试。

## 后端API说明

### 检查目标进度API

**接口**: `GET /api/goal/check?goalType={goalType}`

**参数**:
- `goalType`: 目标类型（weekly 或 monthly）

**返回数据结构** (`GoalCheckResult`):
```json
{
  "code": 200,
  "data": {
    "completed": false,
    "targetValue": 1,
    "actualValue": 1,        // 注意：字段名是 actualValue
    "progressPercentage": 100,
    "goalType": "weekly",
    "goalId": 1
  }
}
```

## 注意事项

1. **字段名一致性**：后端返回 `actualValue`，前端必须使用相同字段名
2. **进度上限**：当实际进度超过目标值时，进度条显示100%，但实际数值显示真实进度
3. **自动刷新**：每次进入页面都会自动刷新进度，无需手动刷新
