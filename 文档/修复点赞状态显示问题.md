# 修复点赞状态显示问题

## 完成时间
2026-03-11

## 问题描述

用户反馈点赞功能存在以下问题：
1. 点赞操作失败，爱心不变色
2. 只有第一次点击时变色，之后不记住状态
3. 提示"已经点过赞"，但爱心图标仍然是灰色（未点赞状态）

## 问题分析

### 根本原因

1. **前端问题**：
   - `userId` 获取方式不正确，导致传递给后端的是 `NaN`
   - 后端返回 400 错误，点赞失败
   - 没有实现乐观更新和错误回滚机制

2. **后端问题**：
   - 查询打卡记录时没有返回 `hasLiked`、`likeCount`、`commentCount` 字段
   - 前端无法知道当前用户是否已经点过赞
   - 每次刷新页面都显示未点赞状态

## 解决方案

### 1. 前端修复

#### 1.1 修复 userId 获取方式

**文件**: `couple-fitness-weapp/components/custom/interaction-bar/index.js`

**问题代码**:
```javascript
currentUserId: storage.getUserId()
```

**修复后**:
```javascript
const userInfo = wx.getStorageSync('userInfo');
const userId = userInfo ? userInfo.userId : null;
```

**原因**: `storage.getUserId()` 可能返回非数字值，导致后端接收到 `NaN`。

#### 1.2 实现乐观更新（Optimistic Update）

**修复内容**:
- 点击时立即更新 UI（切换爱心颜色和点赞数）
- 发送 API 请求
- 如果 API 失败，回滚到原来的状态

**代码示例**:
```javascript
onLike() {
  // 乐观更新 UI
  const newIsLiked = !isLiked;
  const newLikes = isLiked ? this.data.likes - 1 : this.data.likes + 1;
  
  this.setData({
    isLiked: newIsLiked,
    likes: newLikes
  });

  // 发送 API 请求
  api.interactionAPI.like(recordId, currentUserId).then(res => {
    if (res.code === 200) {
      // 成功，保持状态
    } else if (res.msg && res.msg.includes('已经点过赞')) {
      // 已经点过赞，保持点赞状态
      this.triggerEvent('likechange', { 
        recordId: recordId, 
        isLiked: true,
        likeCount: newLikes
      });
    } else {
      // 其他错误，回滚状态
      this.setData({
        isLiked: isLiked,
        likes: this.data.likes - 1
      });
    }
  });
}
```

#### 1.3 处理"已经点过赞"的情况

当后端返回"已经点过赞"的错误时，前端应该：
- 保持点赞状态（红色爱心）
- 不回滚 UI
- 显示友好提示

### 2. 后端修复

#### 2.2 在 Service 层填充互动数据

**文件**: `RuoYi-Vue/ruoyi-admin/src/main/java/com/ruoyi/web/service/impl/CheckInServiceImpl.java`

**添加的方法**:
```java
/**
 * 填充打卡记录的互动数据（点赞数、评论数、是否已点赞）
 */
private void enrichInteractionData(CheckInRecord record, Long currentUserId) {
    if (record == null || record.getRecordId() == null) {
        return;
    }
    
    // 统计点赞数
    int likeCount = interactionMapper.countLikesByRecordId(record.getRecordId());
    record.setLikeCount(likeCount);
    
    // 统计评论数
    int commentCount = interactionMapper.countCommentsByRecordId(record.getRecordId());
    record.setCommentCount(commentCount);
    
    // 检查当前用户是否已点赞
    if (currentUserId != null) {
        Interaction like = interactionMapper.selectLikeByRecordIdAndUserId(
            record.getRecordId(), currentUserId);
        record.setHasLiked(like != null);
    } else {
        record.setHasLiked(false);
    }
}
```

**修改的方法**:
1. `getRecentCheckIns()` - 获取最近打卡记录
2. `getCheckInList()` - 获取打卡记录列表（用于查看伴侣的打卡记录）

**修改内容**:
```java
@Override
public List<CheckInRecord> getRecentCheckIns(Long userId, Integer limit) {
    List<CheckInRecord> records = checkInMapper.selectRecentCheckIns(userId, limit);
    
    // 填充点赞和评论统计数据
    for (CheckInRecord record : records) {
        enrichInteractionData(record, userId);
    }
    
    return records;
}

@Override
public List<CheckInRecord> getCheckInList(CheckInRecord checkInRecord) {
    List<CheckInRecord> records = checkInMapper.selectCheckInList(checkInRecord);
    
    // 获取当前用户ID
    Long currentUserId = null;
    try {
        currentUserId = SecurityUtils.getUserId();
    } catch (Exception e) {
        // 如果获取失败，设置为null
    }
    
    // 填充点赞和评论统计数据
    for (CheckInRecord record : records) {
        enrichInteractionData(record, currentUserId);
    }
    
    return records;
}
```

## 工作流程

### 页面加载时

1. 前端请求打卡记录列表
2. 后端查询打卡记录
3. 后端为每条记录填充互动数据：
   - 统计点赞数
   - 统计评论数
   - 检查当前用户是否已点赞
4. 返回包含完整互动数据的记录
5. 前端根据 `hasLiked` 字段显示正确的爱心状态

### 点赞操作时

1. 用户点击爱心
2. 前端立即更新 UI（乐观更新）
3. 发送点赞 API 请求
4. 后端处理点赞逻辑
5. 返回结果：
   - 成功：保持 UI 状态
   - 已经点过赞：保持点赞状态，显示提示
   - 其他错误：回滚 UI 状态

### 页面返回时

1. 如果有数据变化，触发首页刷新
2. 重新加载打卡记录
3. 显示最新的点赞状态

- 每次查询打卡记录时，都会查询互动数据
- 对于每条记录，执行 3 次数据库查询：
  1. 统计点赞数
  2. 统计评论数
  3. 检查是否已点赞
