# 创建成就表说明

## 问题
数据库中缺少 `achievement` 表，导致成就页面无法加载数据。

## 解决方案

### 方法 1：使用 Navicat 或其他数据库工具

1. 打开 Navicat 或其他 MySQL 数据库管理工具
2. 连接到 `ruoyi-vue-pro` 数据库
3. 打开 SQL 查询窗口
4. 复制并执行以下 SQL：

```sql
-- 成就表 SQL 创建脚本
-- 用于存储用户解锁的成就徽章

CREATE TABLE IF NOT EXISTS achievement (
    achievement_id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '成就ID',
    user_id BIGINT NOT NULL COMMENT '用户ID',
    badge_type VARCHAR(50) NOT NULL COMMENT '徽章类型',
    unlocked_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '解锁时间',
    UNIQUE KEY unique_achievement (user_id, badge_type) COMMENT '用户+徽章类型唯一索引',
    INDEX idx_user_id (user_id) COMMENT '用户ID索引',
    INDEX idx_badge_type (badge_type) COMMENT '徽章类型索引'
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='成就表';
```

### 方法 2：使用 MySQL 命令行

如果你的系统中安装了 MySQL 命令行工具，可以执行：

```bash
mysql -u root -p ruoyi-vue-pro < achievement-table.sql
```

然后输入数据库密码。

### 方法 3：直接在项目中执行

1. 打开项目根目录下的 `achievement-table.sql` 文件
2. 复制其中的 SQL 语句
3. 在数据库管理工具中执行

## 验证

执行完成后，可以在数据库中验证表是否创建成功：

```sql
SHOW TABLES LIKE 'achievement';
```

或者查看表结构：

```sql
DESC achievement;
```

## 徽章类型说明

创建的表支持以下徽章类型：

- `FIRST_CHECKIN`: 首次打卡
- `STREAK_7`: 连续打卡7天
- `STREAK_30`: 连续打卡30天
- `TOTAL_100`: 累计打卡100次
- `COUPLE_GOAL`: 情侣共同目标（双方各打卡50次以上）

## 完成后

创建表后，重新刷新小程序的成就页面，应该就可以正常显示了。
