# 首页点赞显示逻辑说明

根据截图分析：

### 截图1：首页
- 显示：用户8260的打卡记录
- 点赞状态：灰色爱心 + "点赞"文字
- 说明：`hasLiked = false`，`likeCount = 0`

### 截图2：对方打卡记录页面
- 显示：用户8260的打卡记录（同一条）

## 问题分析

### 场景推测

根据截图，有两种可能的场景：

#### 场景A：当前登录用户是8260（查看自己的记录）
```
首页：
- 显示用户8260自己的打卡记录
- hasLiked = false（自己不能给自己点赞）✅ 正确
- likeCount = 0 ❌ 错误（应该显示对方的点赞数1）

对方打卡记录页面：
- 显示用户8260的打卡记录
- hasLiked = true（当前用户8260给自己点赞？）❌ 不合理
- likeCount = 1 ✅ 正确
```

#### 场景B：当前登录用户是4342（查看对方的记录）
```
首页：
- 显示用户4342自己的打卡记录
- 但截图显示的是用户8260 ❌ 矛盾

对方打卡记录页面：
- 显示用户8260的打卡记录
- hasLiked = true（用户4342给8260点过赞）✅ 正确
- likeCount = 1 ✅ 正确
```

### 最可能的情况

**场景A更合理**：当前登录用户是8260，首页显示自己的记录。

问题在于：
1. 首页的 `likeCount` 显示为0，但实际应该是1
2. 对方打卡记录页面的 `hasLiked` 显示为true，这不合理（自己不能给自己点赞）

## 根本原因

### 首页 API 调用流程

```
前端：checkInAPI.getRecentCheckIns(3)
    ↓
后端：GET /api/checkin/recent?limit=3
    ↓
Controller：
    Long userId = SecurityUtils.getUserId();  // 获取当前登录用户ID（8260）
    checkInService.getRecentCheckIns(userId, limit);
    ↓
Service：
    checkInMapper.selectRecentCheckIns(userId, limit);  // 查询用户8260的打卡记录
    ↓
    enrichInteractionData(record, currentUserId);
        currentUserId = SecurityUtils.getUserId();  // 8260
        检查：用户8260是否给这条记录点过赞
        结果：hasLiked = false ✅
        统计：likeCount = ? ❌ 可能有问题
```

### 对方打卡记录页面 API 调用流程

```
前端：checkInAPI.getCheckInListByUserId(partnerId, params)
    ↓
后端：GET /api/checkin/user/{userId}
    ↓
Controller：
    checkInRecord.setUserId(userId);  // 设置查询的用户ID（8260）
    checkInService.getCheckInList(checkInRecord);
    ↓
Service：
    checkInMapper.selectCheckInList(checkInRecord);  // 查询用户8260的打卡记录
    ↓
    enrichInteractionData(record, currentUserId);
        currentUserId = SecurityUtils.getUserId();  // 当前登录用户
        检查：当前登录用户是否给这条记录点过赞
        结果：hasLiked = true（如果当前用户是8260，这不合理）
        统计：likeCount = 1 ✅
```

## 问题定位

需要检查以下几点：

### 1. 首页的 likeCount 为什么是0？

可能原因：
- `enrichInteractionData` 方法中的 `countLikesByRecordId` 返回0
- 数据库中确实没有点赞记录
- 查询条件有问题

### 2. 对方打卡记录页面的 hasLiked 为什么是true？

如果当前登录用户是8260，查看自己的记录，`hasLiked` 不应该是true。

可能原因：
- 数据库中有8260给自己点赞的记录（不应该存在）
- 查询逻辑有问题

## 调试建议

### 步骤1：确认当前登录用户

在首页和对方打卡记录页面分别打印当前登录用户ID：

```javascript
// 首页
console.log('当前登录用户:', wx.getStorageSync('userInfo'));

// 对方打卡记录页面
console.log('当前登录用户:', wx.getStorageSync('userInfo'));
console.log('查看的用户ID:', this.data.partnerId);
```

### 步骤2：检查数据库

查询 interaction 表，确认点赞记录：

```sql
-- 查询这条打卡记录的所有点赞
SELECT * FROM interaction 
WHERE record_id = {recordId} AND type = 'like';

-- 查询用户8260给自己点赞的记录（不应该存在）
SELECT * FROM interaction 
WHERE record_id IN (SELECT record_id FROM check_in_record WHERE user_id = 8260)
  AND user_id = 8260 
  AND type = 'like';
```

### 步骤3：检查后端日志

在 `enrichInteractionData` 方法中添加日志：

```java
private void enrichInteractionData(CheckInRecord record, Long currentUserId) {
    log.info("填充互动数据 - recordId: {}, currentUserId: {}", 
        record.getRecordId(), currentUserId);
    
    int likeCount = interactionMapper.countLikesByRecordId(record.getRecordId());
    log.info("点赞数: {}", likeCount);
    record.setLikeCount(likeCount);
    
    if (currentUserId != null) {
        Interaction like = interactionMapper.selectLikeByRecordIdAndUserId(
            record.getRecordId(), currentUserId);
        log.info("是否已点赞: {}", like != null);
        record.setHasLiked(like != null);
    }
}
```

## 解决方案

### 方案1：修复 likeCount 统计

确保 `countLikesByRecordId` 方法正确统计所有点赞，不管是谁点的。

### 方案2：防止自己给自己点赞

在点赞接口中添加检查：

```java
@Override
public void likeCheckIn(Long recordId, Long userId) {
    // 获取打卡记录
    CheckInRecord record = checkInMapper.selectCheckInById(recordId);
    if (record == null) {
        throw new ServiceException("打卡记录不存在");
    }
    
    // 检查是否是自己的记录
    if (record.getUserId().equals(userId)) {
        throw new ServiceException("不能给自己的打卡记录点赞");
    }
    
    // ... 其他逻辑
}
```

### 方案3：前端防止导航到自己的打卡记录页面

在"对方打卡记录"页面入口处检查：

```javascript
goToPartnerCheckIns() {
    const userInfo = storage.getUserInfo();
    const partnerInfo = storage.getPartnerInfo();
    
    if (!partnerInfo) {
        wx.showToast({
            title: '暂无配对伴侣',
            icon: 'none'
        });
        return;
    }
    
    // 防止查看自己的记录
    if (partnerInfo.partnerId === userInfo.userId) {
        wx.showToast({
            title: '这是你自己的记录',
            icon: 'none'
        });
        return;
    }
    
    wx.navigateTo({
        url: `/pages/partner-checkins/index?userId=${partnerInfo.partnerId}`
    });
}
```

## 总结

根据截图分析，主要问题是：
1. 首页的 `likeCount` 显示为0，应该显示1
2. 需要确认当前登录用户是谁
3. 需要检查数据库中的点赞记录
4. 需要防止自己给自己点赞

建议按照调试步骤逐步排查问题。
