# 后端服务启动检查清单

## 问题现象
前端调用后端API时返回 **404 Not Found** 错误：
- `/api/checkin/statistics/user` - 404
- `/api/checkin/statistics/partner` - 404  
- `/api/checkin/recent` - 404

## 原因分析
404错误说明后端服务器没有找到这些接口，可能的原因：
1. ❌ 后端服务没有启动
2. ❌ 后端服务启动失败
3. ❌ 后端服务端口不是8080
4. ❌ 接口路径配置错误

## 检查步骤

### 1. 检查后端服务是否启动

在命令行中执行：
```bash
cd RuoYi-Vue/ruoyi-admin
mvn spring-boot:run
```

或者在IDEA中：
- 找到 `RuoYiApplication.java` 文件
- 右键 → Run 'RuoYiApplication'

### 2. 查看启动日志

启动成功应该看到类似的日志：
```

( ( )\___ | '_ | '_| | '_ \/ _` | \ \ \ \

 =========|_|==============|___/=/_/_/_/
 :: Spring Boot ::        (v2.5.15)

...
Started RuoYiApplication in X.XXX seconds
```

### 3. 检查端口是否正确

查看启动日志中的端口信息：
```
Tomcat started on port(s): 8080 (http)
```

如果端口不是8080，需要修改前端的 `BASE_URL`：
- 文件：`couple-fitness-weapp/utils/request.js`
- 修改：`const BASE_URL = 'http://localhost:端口号';`

### 4. 测试接口是否可访问

在浏览器或Postman中访问：
```
http://localhost:8080/api/checkin/statistics/user
```

**预期结果**：
- 如果返回401：说明接口存在，但需要登录（正常）
- 如果返回404：说明接口不存在（有问题）
- 如果无法连接：说明服务没启动（有问题）

### 5. 检查数据库连接

查看 `application.yml` 配置：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/数据库名?useUnicode=true&characterEncoding=utf8&zeroDateTimeBehavior=convertToNull&useSSL=true&serverTimezone=GMT%2B8
    username: root
    password: 你的密码
```

确保：
- MySQL服务已启动
- 数据库已创建
- 用户名密码正确

### 6. 检查Controller是否被扫描

确认 `CheckInController.java` 的包路径：
```java
package com.ruoyi.web.controller;

@RestController
@RequestMapping("/api/checkin")
public class CheckInController extends BaseController {
    // ...
}
```

## 快速修复方案

### 方案1：重启后端服务
```bash
# 停止当前运行的服务（Ctrl+C）
# 重新启动
cd RuoYi-Vue/ruoyi-admin
mvn clean spring-boot:run
```

### 方案2：检查Maven依赖
```bash
cd RuoYi-Vue
mvn clean install
```

### 方案3：使用测试登录（跳过后端）
如果后端暂时无法启动，可以使用前端的测试登录页面：
- 访问：`/pages/login/test-login`
- 这个页面可以模拟登录，不依赖后端

## 验证修复

后端启动成功后，在微信开发者工具中：
1. 清除缓存
2. 重新编译
3. 重新登录
4. 查看控制台，应该不再有404错误

## 常见错误

### 错误1：端口被占用
```
Port 8080 was already in use
```
**解决**：
- 关闭占用8080端口的程序
- 或修改 `application.yml` 中的端口号

### 错误2：数据库连接失败
```
Could not create connection to database server
```
**解决**：
- 启动MySQL服务
- 检查数据库配置

### 错误3：Maven依赖下载失败
```
Could not resolve dependencies
```
**解决**：
- 检查网络连接
- 配置Maven镜像源（阿里云）
